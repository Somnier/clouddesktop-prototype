## 1. 这页真正要解决什么问题

TERM-04 承接的是“当前教室这一轮怎么部署”，不是一个额外被命名、被保存的新对象。

这页上真正存在的，是一组现场工程师能够直接理解和直接操作的设置项：

1. 这次到底要部署到哪些终端。
2. 这些终端按什么顺序处理。
3. 终端用途怎么和这轮部署一起写入。
4. 座位号怎么按顺序分配。
5. IP 地址怎么按顺序分配。
6. 机器名怎么按顺序分配。
7. 服务器地址要不要一并写入。
8. 这次要下发哪些桌面。
9. 默认启动哪个桌面。
10. 这次写入时是否整体覆盖已有桌面内容。

因此 TERM-04 不应该把这些设置项再包装成一个新的实体对象。原型里需要表达的是：工程师在这一页逐项设置，系统根据这些设置项给出逐台终端的写入结果预览，确认后再开始真正分发。

这页还必须把流程闭环补完整：

1. 先在当前母机上完成本机准备，包括机器名、座位号、终端用途、IP、网络配置、服务器地址和本机桌面。
2. 再选这次要部署到哪些终端，并调整顺序。
3. 再确定这轮终端用途、座位号、IP、机器名等写入规则。
4. 再从当前母机已经准备好的本机桌面里勾选这次要下发的一个或多个桌面。
5. 回到管理系统核对每台终端将被写成什么。
6. 校验 IP、机器名等冲突项。
7. 开始分发。
8. 停留在结果页，方便现场继续看哪台成功、哪台失败，并决定是否在当前任务页继续处理异常。
9. 确认这轮结果后，再返回 TERM-03 或转入其他流程。

## 2. 这页与其他页面的关系

1. TERM-04 从 [TERM-03_母机工作页.md](./TERM-03_母机工作页.md) 的“单教室部署”进入。
2. TERM-04 的起始阶段先承接当前母机的本机准备；这一步属于单教室部署主线的一部分。当前母机如需处理本机桌面，就在这一页的当前母机准备区点击 [修改本机桌面]；终端会重启进入当前母机的本机桌面处理，在那里可新建桌面、复制已有桌面，或进入桌面编辑；完成这次桌面处理并再次重启后回到 TERM-04 的当前母机准备区，再继续后续步骤；“选择部署桌面”阶段只从当前母机已有本机桌面里选择，不重复展开桌面制作动作。TERM-01 的本机入口继续保留给教师终端单机部署、晚补单机和其他非批量场景。
3. 当前教室有哪些终端属于受管集合、这次勾选了哪些终端、受控终端怎样显示当前任务，以 [云桌面批量部署范围与终端状态同步机制说明.md](../05_云桌面事实/云桌面批量部署范围与终端状态同步机制说明.md) 为准。
4. 如果当前母机准备阶段点击 [修改本机桌面] 进入桌面处理，完成并再次重启后仍回 TERM-04 的当前母机准备区；部署桌面选择阶段直接进入已有桌面选择、默认启动选择和写入确认。
5. TERM-04 执行期间，受控终端停留在 [TERM-01_管理系统首页.md](./TERM-01_管理系统首页.md) 的受控任务屏，只看自己的当前进度。
6. TERM-04 的设置、执行和结果停留都留在 TERM-04；只有当用户主动离开当前任务页时，才返回 TERM-03。若软件重进时当前任务上下文仍在，应直接回到 TERM-04，而不是先落 TERM-03。

## 3. 当前页依赖的数据层

### 3.1 当前教室上下文

1. 当前教室名称。
2. 当前母机身份。
3. 当前受管终端集合。
4. 最近一次确认后的成员总数。
5. 最近刷新时间。

### 3.2 当前页面上的设置项

1. 当前母机自己的机器名、座位号、终端用途。
2. 当前母机自己的 IP、网络配置、服务器地址。
3. 当前母机当前可部署的桌面集合与桌面准备状态。
4. 当前勾选了哪些终端。
5. 这些终端当前按什么顺序排列。
6. 终端用途写入规则。
7. 座位号分配规则。
8. IP 地址分配规则。
9. 机器名分配规则。
10. 服务器地址。
11. 这次要下发的桌面集合。
12. 默认启动桌面。
13. 还原模式。
14. 数据盘设置。
15. 当前桌面内容处理方式。

### 3.3 当前母机本机桌面状态

1. 当前母机已有桌面列表，至少展示桌面名称、时间和备注。
2. 当前勾选的部署桌面集合。
3. 默认启动桌面。
4. 数据盘设置和当前内容处理方式。

### 3.4 执行前校验与执行结果

1. 当前生效网络与要写入的网络配置对照。
2. IP 地址冲突检查。
3. 机器名冲突检查。
4. 必填项缺失检查。
5. 本地分发进度。
6. 失败终端和失败原因。

## 4. 页面上的关键交互规则

### 4.1 TERM-04 的起点是当前母机准备，不是直接框选学生终端

正式基线已经把“母机终端桌面制作”和“单教室终端批量部署”并列放在单教室部署主线上，因此 TERM-04 的起点不应直接跳到学生终端范围设置。

更准确的表达应该是：

1. 先在 TERM-04 起始阶段完成当前母机自己的机器名、座位号、终端用途、IP、网络配置、服务器地址和本机桌面准备。
2. 当前母机准备完成后，再勾选这次要部署到的终端。
3. 如有需要，拖动顺序，决定后续座位号、IP、机器名按什么顺序分配。
4. 再设置座位号、IP、机器名、服务器地址等规则。
5. 系统实时把这些设置项展开成逐台终端的写入预览。

如果后续研发要把这套规则缓存下来，供下次同一教室复用，那是增强能力，不是当前原型必须引入的新对象。

### 4.2 终端选择与排序

这一步至少要支持下面几种操作：

1. 在终端列表中逐台勾选。
2. 全选当前列表。
3. 清空当前勾选。
4. 按当前工作 MAC 排序；若当前 IP、当前机器名或当前座位号已经存在，也可参与排序，但不在这里写死默认排序字段。
5. 手动取消某几台终端。
6. 对已勾选终端上下拖动排序。
7. 只要终端列表需要人工勾选，就应支持分页；分页策略按当前终端规模决定，不在这里写死。

这里有一条必须说清的事实：单教室初次部署时，这批终端可能根本还没有完成初始批量设置，因此页面不应假设当前已经存在可靠的“教师终端/学生终端”快捷筛选入口。更稳妥的做法是提供全选、字段排序和逐台勾选，再把终端用途作为这轮部署写入的一部分处理。

还要补一条显示原则：单教室初次部署时，这批终端很可能只有当前工作 MAC 是稳定且可直接核对的字段。因此这一步的线框图应把当前工作 MAC 放在第一列；当前 IP、当前机器名、当前座位号如果已经存在，就作为补充信息一起显示；没有就不强行占位。第二次以后终端字段更完整时，可以显示更多辅助信息，但那不是这页主图的默认前提。

### 4.3 终端用途、座位号、IP 和机器名要一起写入

这一步不只是“写一个 IP 段”。至少应允许用户：

1. 设置终端用途写入方式，例如默认学生终端，并允许在确认页逐台改成教师终端或其他用途。
2. 设置 IP 地址网段前缀和起始编号，并按当前顺序自动递增。
3. 设置机器名前缀和起始编号，并按当前顺序自动递增。
4. 设置座位号规则，并说明 A 列有多少台、B 列有多少台，后续按当前顺序自动排座位。

也就是说，终端用途、座位号、IP 和机器名都属于这轮部署的同一批写入结果。顺序本身又是后续很多写入项的基础，所以排序必须是清晰可操作的，而不是藏在一句抽象总结里。

另外，当前母机自己的机器名、IP、座位号、用途和本机桌面，应在进入 TERM-04 后先于流程起始阶段直接完成准备，再继续批量部署。

### 4.4 部署桌面只从当前母机已有本机桌面里选择

这一步不再重复展开桌面制作动作。更稳妥的规则是：

1. 本次要下发的桌面集合可以多选，但来源只能是当前母机已经准备好的本机桌面。
2. 默认启动桌面必须从这次要下发的桌面集合里选择 1 个。
3. 如果发现当前母机还缺目标桌面，或某个桌面内容还要调整，应先回到当前母机准备区处理本机桌面，再继续批量部署。
4. 这一步只负责确定“这轮批量部署要带哪些已有桌面”和“默认启动桌面是谁”。
5. 只要桌面列表需要人工勾选，就应支持排序和分页；可排序字段和分页策略按桌面数量与当前场景决定，不写死。

### 4.5 当前母机桌面修改留在准备阶段完成

1. 当前母机的桌面修改属于当前母机准备的一部分，不再在“选择部署桌面”之后重复展开一套桌面制作步骤。
2. 当前母机准备区可以复用 [TERM-01_管理系统首页.md](./TERM-01_管理系统首页.md) 的本机桌面字段组和动作心智，但在 TERM-04 里它服务的是“先准备好当前母机，再继续整批部署”。
3. 因此 TERM-04 的批量主线保持为“准备当前母机 -> 选终端与规则 -> 选已有桌面 -> 确认写入 -> 分发”。

### 4.6 选好已有桌面后直接进入写入确认

1. 当前母机桌面已经准备好的前提下，选择完部署桌面就应直接进入写入确认页。
2. 这里直接衔接逐台写入确认，不额外增加历史遗留的中间步骤。
3. 软件重进时，如果当前任务上下文仍在，应直接回到 TERM-04 当前步骤，而不是落到其他中间页。

### 4.7 执行前必须同时校验 IP 和机器名

当前页不能只校验 IP 冲突，还必须校验机器名冲突。

只要还存在下面任一问题，就不能开始分发：

1. IP 地址冲突。
2. 机器名冲突。
3. 座位号规则没排完。
4. 仍然没有选中任何终端。
5. 必填字段为空。

## 5. 页面字段定义

| 字段 | 含义 | 来源 | 何时可显示 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| 这次要部署到的终端 | 当前页勾选中的终端 | 当前教室上下文 + 当前页设置项 | 进入 TERM-04 后 | 来源于当前页勾选，不是新对象 |
| 当前顺序 | 当前勾选终端在这一页里的处理顺序 | 当前页设置项 | 勾选后 | 座位号、IP、机器名默认按这个顺序展开 |
| 保持现状终端 | 当前教室里这轮不执行的终端 | 当前页设置项 | 勾选后 | 需要同时说明本次不修改和沿用了什么当前内容 |
| 座位号规则 | 按列数和顺序自动排座位的规则 | 当前页设置项 | 设置后 | 例如 A 列 8 台、B 列 8 台 |
| IP 地址规则 | 按网段前缀和起始编号自动分配的规则 | 当前页设置项 | 设置后 | 执行前要检查冲突 |
| 机器名规则 | 按前缀和起始编号自动分配的规则 | 当前页设置项 | 设置后 | 执行前也要检查冲突 |
| 终端用途写入规则 | 这轮部署准备给终端写入什么用途 | 当前页设置项 | 设置后 | 可默认批量设置，也可在确认页逐台改 |
| 部署桌面集合 | 这次部署要下发的一个或多个桌面 | 当前母机本机桌面状态 | 进入桌面选择页后 | 允许多选 |
| 默认启动桌面 | 这次部署后终端默认进入哪个桌面 | 当前页设置项 + 当前母机本机桌面状态 | 选中桌面后 | 必须来自部署桌面集合 |
| 写入信息确认页 | 逐台核对最终要写入什么 | 当前页设置项 + 桌面选择结果 | 选好部署桌面后 | 这是分发前最后确认页 |
| 校验结果 | 当前还存在哪些冲突和缺失 | 执行前校验与执行结果 | 写入信息展开后 | IP 和机器名冲突都要拦截 |
| 分发结果 | 当前分发进行到哪一步 | 执行前校验与执行结果 | 发起后 | 结果页要停留给现场核对 |

## 8. 页面结论

1. TERM-04 的第一步是当前母机准备，当前母机自己的机器名、网络和本机桌面都在这里处理，然后才进入整批部署。
2. 终端用途、终端顺序、座位号规则、IP 规则、机器名规则，都应该作为当前页上的直接设置项出现；页面不应依赖预先存在的教师终端或学生终端快捷筛选。
3. “选择部署桌面”这一步只从当前母机已有本机桌面里多选目标桌面，并继续确认默认启动桌面和逐台写入结果。
4. 默认启动桌面必须从本次下发桌面集合里选择 1 个。
5. 分发前必须同时校验 IP 冲突和机器名冲突。
6. 分发结果页必须停留给现场核对，不能直接跳过。
7. “保持现状”不应只给一个抽象标签；默认应同时告诉用户这一项本次不修改，以及它实际沿用了什么内容。
8. TERM-04 的批量主线现在收束为“准备当前母机 -> 选终端与规则 -> 选已有桌面 -> 确认写入 -> 分发”。
