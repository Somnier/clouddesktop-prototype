## 1. 这页真正要解决什么问题

TERM-03 不是接管识别页，也不是 TERM-04、TERM-05、TERM-07 的任务详情页。它是母机接管成功后的“当前教室工作台”。

这页真正要回答的是 4 件事：

1. 当前正在管理哪间教室，当前母机是谁。
2. 当前教室的受管终端现在是什么状态，哪些终端有异常。
3. 当前教室最近测试结果怎样，当前有没有需要立即关注的问题。
4. 现在能从这里进入哪些教室级动作，例如单教室部署、教室维护、考试场景、补录终端和导出清单。

TERM-03 只负责“当前教室工作面”。一旦用户已经进入 TERM-04、TERM-05、TERM-07，对应任务的设置、执行、取消和结果停留都应留在各自任务页里，不再把 TERM-03 写成任务锁定态、任务返回态或所谓收口态。

## 2. 这页与其他页面的关系

1. TERM-03 从 [TERM-02_母机接管页.md](./TERM-02_母机接管页.md) 接管成功后进入。
2. TERM-03 是 [TERM-04_单教室部署流程.md](./TERM-04_单教室部署流程.md)、[TERM-05_教室维护流程.md](./TERM-05_教室维护流程.md)、[TERM-07_考试桌面流程.md](./TERM-07_考试桌面流程.md) 的教室级入口。
3. TERM-03 不负责单终端故障处理的主入口；一键替换、一键重置仍属于 TERM-01 / TERM-06 的单机视角。
4. 正常管理路径里，接管成功后直接进入 TERM-03；不会先回 TERM-01 再进来。
5. 软件退出后重新进入，如果本机仍保留当前母机对当前教室的管理关系，系统应直接恢复：当前有教室级任务就回到对应任务页的当前步骤；当前没有进行中的教室级任务就回到 TERM-03。
6. TERM-03 只在“当前没有进行中的教室级任务，但这层管理关系仍有效”时作为恢复落点；不会先经过 TERM-01 的中间页。
7. TERM-04、TERM-05、TERM-07 负责当前任务的详细设置、执行和结果页；TERM-03 只负责入口和当前教室摘要，不吞并这些页的正文，也不承接任务结果收口。
8. 当前教室里的异常终端仍然属于当前教室受管集合，应继续显示在 TERM-03 的终端列表里；异常不等于脱离教室范围，也不等于必须由某一轮任务来定义它。
9. TERM-03 上的教室级动作前提要按动作拆开判断：单教室部署、教室维护、考试场景各自展示自己的成立条件、待补项和下一步入口。

## 3. 当前页依赖的数据层

### 3.1 当前教室上下文

1. 当前教室名称。
2. 当前受管终端集合。
3. 当前母机身份。
4. 最近一次确认后的成员总数。
5. 最近刷新时间。

### 3.2 当前教室当前状态

1. 当前在线数。
2. 当前异常摘要。
3. 最近测试结果摘要。
4. 当前同步状态。
5. 当前教室是否存在需要补录或补充信息的终端。
6. 当前母机本机准备是否已齐备。

### 3.3 当前教室动作可用性

1. 当前教室级入口是否可直接发起。
2. 当前补录终端动作是否可用。
3. 当前导出清单动作是否可用。
4. 当前测试与告警查看动作是否可用。

### 3.4 终端清单级数据

1. 座位号与机器名。
2. 当前 IP 地址。
3. 当前工作 MAC。
4. 终端用途。
5. 在线状态。
6. 当前状态或异常摘要。
7. 当前任务关系和当前任务状态。
8. 异常摘要。

## 4. 页面字段定义

| 字段 | 含义 | 来源 | 何时可显示 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| 当前教室名称 | 本次接管确认后的当前教室名称 | 当前教室集合 | 进入 TERM-03 后 | 可编辑，但不能脱离当前受管集合 |
| 受管终端数 | 当前已纳入本教室受管集合的终端数量 | 当前教室集合 | 进入 TERM-03 后 | 补录成功后会变化 |
| 当前在线数 | 当前受管终端集合中当前可达的终端数量 | 当前教室当前状态 | 进入 TERM-03 后 | 这是运行态数据，不是扫描识别数据 |
| 当前异常摘要 | 当前受管集合里的异常汇总 | 当前教室当前状态 | 进入 TERM-03 后 | 没有异常时也应明确显示“当前无异常”或等价结果 |
| 最近测试摘要 | 最近一次当前教室测试结果汇总 | 当前教室当前状态 | 有测试结果后 | 没有结果时应明确为空 |
| 当前教室动作摘要 | 当前可直接进入哪些教室级动作 | 当前教室动作可用性 | 进入 TERM-03 后 | 这里只回答当前能做什么，不展开任务正文 |
| 教室级动作前提摘要 | 单教室部署、教室维护、考试场景各自动作当前能否继续，以及各自还缺什么 | 当前教室当前状态 | 进入 TERM-03 后 | 必须按动作拆开显示，不能混成一个总开关 |
| 导出完整性摘要 | 当前教室终端关键信息是否完整 | 当前教室当前状态 | 导出前 | 优先按“信息完整/待补充”表达，而不是默认写成“部署是否完成” |
| 终端当前状态 | 当前终端在教室里的当前状态 | 终端清单级数据 | 进入 TERM-03 后 | 默认先看当前状态和异常，不默认先看任务关系 |

## 5. 显示规则

### 5.1 TERM-03 默认先展示“当前教室状态”，不是“某轮任务范围”

在没有教室级任务执行时，TERM-03 默认工作态首先要回答的是：

1. 当前教室里有哪些终端。
2. 它们现在是在线、离线还是异常。
3. 当前教室有没有明显风险。

因此默认工作态不应该一上来就展示“这轮勾选多少、未纳入多少、未完成多少”。这些都属于某一轮任务的语义，不属于当前教室默认状态。

### 5.2 异常终端仍然属于当前教室范围，应继续显示在列表里

TERM-03 的终端列表优先服务现场判断。异常终端仍然属于当前教室受管集合，不应因为它异常了，就从默认列表里消失。

默认列顺序建议如下：

1. 座位号与机器名。
2. 当前 IP 地址。
3. 当前工作 MAC。
4. 终端用途。
5. 在线状态。
6. 当前状态或异常摘要。

补充规则：

1. 不再单独列一段“识别顺序”；真正定位一台终端时，应尽量同时给出座位号、机器名、当前 IP 地址和当前工作 MAC。
2. 若某台终端尚未建立稳定座位号，也不应把机器名或 IP 单独伪装成唯一来源，而应保留当前工作 MAC 作为现场确认字段。
3. TERM-03 的默认终端列表不再补“本轮关系 / 当前任务状态”这类任务列；任务相关信息留在 TERM-04、TERM-05、TERM-07 自身的任务页里表达。
4. 补录终端弹层里的人工勾选列表同样应支持排序和分页；可排序字段和分页策略按当次扫描结果规模决定，不写死。

### 5.3 TERM-03 不承接执行中任务或任务结果页

1. 只要用户已经进入 TERM-04、TERM-05、TERM-07，对应任务的设置、执行、取消和结果停留都留在对应任务页。
2. 软件重进时，如果当前教室级任务上下文仍在，应直接回到对应任务页，而不是先回 TERM-03。
3. TERM-03 不再定义“任务锁定态”“任务返回态”或“收口态”。

### 5.4 TERM-03 上的教室级动作前提要按动作拆开判断

1. TERM-03 至少同时承接单教室部署、教室维护、考试场景、补录终端、导出清单这些动作；页面要分别显示它们各自的成立条件和下一步入口。
2. 单教室部署遵循“先行设置当前母机，再行单教室终端批量部署”的基线顺序。点击 [单教室部署] 后，TERM-04 先完成当前母机自己的机器名、座位号、终端用途、IP、网络配置、服务器地址和本机桌面准备，再继续批量部署阶段。
3. 教室维护要按维护类型判断。桌面更新类维护要求当前母机已经准备好目标本机桌面；TERM-05 的批量步骤只选择这些已有桌面，不再把桌面制作并进批量步骤。批量改终端 IP、批量改服务器地址这两类轻量配置，只要求当前母机能够和受管终端通信，不要求当前母机已有桌面。
4. 考试场景要求当前母机能够继续管理当前教室；考试桌面也应先在当前母机本机桌面里准备好，TERM-07 再选择已有桌面并继续写考试启动设置和考后恢复。
5. 补录终端和导出清单继续按当前教室识别结果与字段完整性判断，不跟随上述三条批量流程共用同一套门槛。

### 5.5 导出提示优先按“关键信息是否完整”表达

正式基线已经明确：系统兼容“先导出清单并创建教室、后制桌面”。因此导出提示不能默认写成“部署是否完成”。

更稳妥的表达是：

1. 当前哪些终端关键信息已完整。
2. 哪些终端的用途、座位号、机器名或其他导出字段仍待补充。
3. 如果当前仍有关键字段缺失，应继续列出原因分类，并直接指向对应动作下一步需要补齐的内容。

## 8. 跨页联动说明

1. TERM-03 发起的部署、维护、考试任务，必须同步到受控终端的 TERM-01 状态展示里。
2. TERM-03 的导出动作和补录动作可以保留为本页弹层，不必额外拆独立页面。
3. TERM-03 默认展示的是整间教室的当前状态；任务设置、执行和结果停留都留在 TERM-04、TERM-05、TERM-07 自身。
4. TERM-03 不再承接“查看待继续处理终端”这类任务结果收口视图。
5. TERM-03 只负责“当前教室工作面”，不反向吞并单终端首次进入、故障处理等本机级场景。
6. 如果软件重进时当前没有进行中的教室级任务，系统直接回到 TERM-03；如果当前已有任务，则直接回到对应任务页，而不是先经过 TERM-01 的中间页。
