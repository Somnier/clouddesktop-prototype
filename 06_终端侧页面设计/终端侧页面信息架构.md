## 4. 终端侧通用布局原则

以下布局规则适用于 TERM-01 到 TERM-07 所有终端侧页面：

1. 所有页面在 1920×1080 分辨率下不出现页面级滚动条。如果列表、表格或卡片区域内容超出可视区域，只允许该区域内部滚动，不将操作按钮藏在页面滚动条底部。
2. 步骤条（stepBar）和页面标题始终固定在页面顶部可见区域。底部操作按钮始终固定在页面底部可见区域。中间内容区作为唯一可滚动区域。
3. 表单标签与输入框之间使用紧凑间距（gap: 12px），不使用 space-between 拉满整行。标签固定宽度，输入框自适应剩余空间。
4. 受控终端任务屏为居中状态显示，内容始终在一屏内，不需要滚动。

## 6. 终端侧主流程

### 6.1 单教室初次交付

TERM-01 管理系统首页 -> TERM-02 母机接管页 -> TERM-03 母机工作页 -> TERM-04 单教室部署流程 -> TERM-03 导出教室终端清单

补充说明：

1. 这条主线不以服务器存在为前提。
2. 若已知服务器地址，可在部署阶段一并写入终端业务配置。
3. 导出教室终端清单依赖的是本地结果，不依赖服务器是否已创建教室。
4. 如果软件在管理过程中退出，系统恢复时直接回到对应页面：若当前已有教室级任务，应直接返回对应任务页；只有当前没有进行中的教室级任务时才回 TERM-03。

### 6.2 交付后维护

TERM-01 管理系统首页 -> TERM-02 接管已有教室 -> TERM-03 母机工作页 -> TERM-05 教室维护流程

补充说明：

1. 教室维护先解决当前教室本地结果。
2. 如果当前教室已经配置服务器且服务器可用，桌面更新后可继续上传到服务器。
3. 上传影响的是平台侧查看和跨教室复用，不改变当前教室本地是否已经更新完成。
4. 维护任务执行中和结果停留都留在 TERM-05 自身；TERM-03 只在用户主动离开任务页后继续承接当前教室工作面。

### 6.3 考试前设置与考后恢复

TERM-03 母机工作页 -> TERM-07 考试桌面流程

补充说明：

1. 考试桌面仍然是终端侧教室级流程的一部分，不是另一套独立系统。
2. 考试前设置和考后恢复都属于批量终端动作。

### 6.4 单终端故障处理

TERM-01 管理系统首页 -> TERM-06 故障处理流程

补充说明：

1. 一键替换和一键重置都属于单终端入口。
2. 一键重置依赖服务器恢复源；一键替换依赖服务器登记和当前网络可达。

## 7. 为什么要同时定义母机页、受控任务屏和 TERM-06 的边界

这里的“同时定义”不表示三者属于同一条执行中的流程，也不表示页面之间可以随意跳转。

真正需要一起定义的是下面这组边界：

1. 母机页负责整批任务视角。它要看整间教室、这轮勾选终端、整批进度和整批结果。
2. 受控任务屏负责本机视角。它要让某一台终端知道自己当前受谁控制、自己有没有被纳入、自己现在执行到哪一步。
3. TERM-06 负责脱离整批任务后的单机视角。它处理的是替换、重置和网络修正这类单机问题，而不是整批任务。

因此要一起定义的不是“它们如何并行工作”，而是：

1. 母机页和受控任务屏属于同一轮批量任务的两个视角。
2. 受控执行中，受控终端不能自行跳到 TERM-06。
3. TERM-06 只在终端未受控、当前任务已中断、当前任务已取消，或终端已经明确脱离整批任务后才成立。

后续页面写法必须同时记住这层关系，避免把 TERM-06 写成任务执行中的旁路入口，也避免把单机排障误写成教室级任务页面。

## 8. 三类教室级流程的共同骨架

TERM-04、TERM-05、TERM-07 分别承接部署、维护、考试三类教室级任务，但只要页面里同时涉及桌面内容和批量终端处理，它们都应沿用同一条基本骨架：

1. 先确认当前母机本机准备是否齐备；单教室部署缺项时在 TERM-04 起始阶段直接补齐本机信息、网络与服务器、本机桌面；维护和考试如果缺桌面或要改已有桌面，则在当前步骤点 [修改本机桌面]，终端重启进入当前母机的本机桌面处理；在那里可新建桌面、复制已有桌面，或进入桌面编辑；完成这次桌面处理并再次重启后，系统回到原任务页当前步骤，再进入批量步骤。
2. 再确定这轮要处理哪些终端。
3. 如有需要，调整顺序、范围，并通过字段排序、逐台勾选和已勾选终端拖动完成终端选择。
4. 按场景确定要处理的桌面：04、05、07 都只从当前母机已经准备好的本机桌面里选择一个或多个待下发桌面。
5. 再逐台确认要写入什么。
6. 最后开始批量执行并停留在结果页。
7. 执行中和结果停留都留在各自任务页；如果软件重进且当前任务上下文仍在，也应直接回到对应任务页。

这里有一条必须固定的边界：本机桌面处理和批量桌面选择是两层动作。当前母机的桌面新建、复制、编辑属于本机桌面准备；TERM-04、TERM-05、TERM-07 的批量步骤只负责从已有本机桌面里选择这轮要下发的桌面集合。后续页面不能再把这两层混成同一个步骤。

## 9. 当前终端侧页面的稳定口径

1. 页面正文继续稳定使用“母机”“受控终端”等机制术语。
2. 按钮和入口名可以保留更直接面向用户的表达，例如“进入教室管理模式”而不是“进入母机工作模式”。
3. 终端业务配置至少包括座位号、机器名、网络配置和服务器地址；页面需要区分配置值和当前生效值。
4. 终端列表和结果页应优先显示当下真实存在、且对当前任务最有价值的识别字段：已有座位号和机器名时可以前置；首次部署只剩当前工作 MAC 时，就以 MAC 为主，当前 IP、机器名、座位号有则补充、无则不强行占位。
5. 终端本地完成判断和服务器后续同步是两条并行线，不互相替代。
6. TERM-01 的 [本机桌面] 只处理当前终端自己的本机桌面：可以新建、复制、编辑、设为默认启动，也可以调整当前终端的还原设置；页面选择按单桌面操作展开，用于查看当前选中桌面并执行当前终端上的桌面动作。进入桌面编辑后，终端重启进入当前选中桌面的桌面环境；完成这次桌面处理并再次重启后，系统回到发起它的页面。
7. [结束本次管理] 只属于 TERM-03 的当前教室工作页；TERM-04、TERM-05、TERM-07 如果要中止流程，使用各自的取消当前任务动作，任务结束或取消后再回 TERM-03 结束本机对当前教室的母机管理。
8. 批量任务里的机器名、座位号、终端用途、网络配置、服务器地址、桌面集合、默认启动项和还原方式，与 TERM-01 单机页里处理的是同一组业务字段；区别只在整批写入还是单台处理。
9. 桌面相关批量流程现在统一收束为纯批量流：单教室部署在当前母机准备阶段完成本机桌面处理；教室维护和考试如需改桌面，则在桌面选择步骤使用单独的 [修改本机桌面] 按钮，处理完成并再次重启后回到原步骤；真正的批量步骤只负责选择已有本机桌面、预览写入和批量执行。
