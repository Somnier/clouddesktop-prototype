## 1. 这页真正要解决什么问题

TERM-01 不是普通入口墙，而是任何终端进入管理系统后的第一落点。它必须同时覆盖 3 类可见状态和 1 条恢复规则：

1. 刚完成底层安装、还没有教室关系和桌面内容的终端。
2. 已完成基本配置，但当前没有被任何母机接管的终端。
3. 当前已被母机接管的受控终端。
4. 如果本机仍保留当前母机对某间教室的管理关系，软件重进后系统要直接恢复到对应页面，而不是停在一个中间首页。

因此 TERM-01 既要回答“这台终端现在是谁”，也要回答“这台终端现在归谁控制、还能做什么”。

## 2. 这页与其他页面的关系

1. 点击“进入教室管理模式”后先进入 [TERM-02_母机接管页.md](./TERM-02_母机接管页.md)，而不是直接进入 [TERM-03_母机工作页.md](./TERM-03_母机工作页.md)。
2. 接管成功后的正常工作路径应直接进入 [TERM-03_母机工作页.md](./TERM-03_母机工作页.md)，而不是先回 TERM-01。
3. 如果软件重新进入管理系统时，本机仍保留当前母机对当前教室的管理关系，系统应直接恢复到对应页面：当前有教室级任务就回到对应任务页的当前步骤；当前没有进行中的教室级任务就回到当前教室工作页。TERM-01 不额外停留恢复中间页。
4. 点击“故障处理”进入 [TERM-06_故障处理流程.md](./TERM-06_故障处理流程.md)。
5. TERM-01 不负责设置终端用途；终端用途由部署或维护流程写入，首页只负责回显结果。
6. “本机信息”“网络与服务器”“本机桌面”都不是模糊按钮，而是 TERM-01 下的本机级二级页；前者负责机器名、座位号、终端用途和硬件信息的单机准备或回显，中者负责本机 IP、DNS、网关和服务器地址等容错配置，后者负责当前终端已有桌面的单机准备、复制、编辑、默认启动设置和还原设置。
7. 当前终端一旦进入受控态，TERM-01 就不再是自由导航首页，而应变成“受控任务屏”。
8. 如果当前母机不可达且当前任务无法继续，TERM-01 必须承接异常脱管倒计时和自动退出结果。

## 3. 当前页依赖的数据层

### 3.1 本机实时数据

1. 机器名。
2. 当前 IP 地址。
3. 当前工作 MAC。
4. 当前网络连通状态。
5. 当前与服务器连通状态。

### 3.2 本机业务配置

1. 终端用途标记；首次装机后可为空，后续由部署或维护流程写入，当前至少会出现学生用途和教师用途，但字段本身允许继续扩展。
2. 若已建立则显示所属教室和座位号。
3. 机器名配置。
4. 网络配置，包括 IP、子网掩码、网关、DNS。
5. 服务器地址。
6. 已安装桌面摘要。

### 3.3 当前控制状态

1. 未接管。
2. 当前母机。
3. 已受管但不是母机。
4. 受控关系异常结束。

### 3.4 当前任务状态

1. 当前是否存在部署、维护、考试、故障处理等正在执行的任务。
2. 当前任务是否由母机发起。
3. 当前任务是否已完成本机计划内容并得到当前任务结果确认。
4. 当前任务是否中断待处理。
5. 当前任务是否已取消。

## 4. 页面字段定义

| 字段 | 含义 | 来源 | 何时可显示 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| 机器名 / 当前IP地址 / 当前工作MAC | 当前终端首屏直接展示的识别字段 | 本机实时数据 | 始终可显示 | 尽量同时显示机器名、当前 IP 地址和当前工作 MAC；空间不足时至少保留“当前 IP 地址 + 当前工作 MAC” |
| 当前工作 MAC | 当前实际联网使用的 MAC | 本机实时数据 | 始终可显示 | 现场核对优先字段 |
| 终端用途 | 当前终端被写入的用途标记 | 本机业务配置 | 已标记后显示 | 未设定时直说"未设定"，不把用途枚举写死在首页结构里 |
| 网络配置摘要 | 当前生效网络与已保存业务配置的关系 | 本机实时数据 + 本机业务配置 | 网络字段可读时 | 至少区分“当前生效 IP”和“业务配置中的 IP”，两者不一致时要显式提示 |
| 服务器地址状态 | 是否已配置服务器地址 | 本机业务配置 | 始终可显示 | 一键替换等依赖它 |
| 桌面可用状态 | 当前是否已有可进入的桌面 | 本机业务配置 | 始终可显示 | 没有桌面时“进入桌面”禁用 |
| 当前控制状态 | 未接管 / 当前母机 / 已受管 / 异常脱管中 | 当前控制状态 | 始终可显示 | 决定首页主动作和可用性 |
| 当前母机识别 | 当前控制本机的母机 | 当前控制状态 | 已受管时显示 | 受控终端必须知道自己受谁控制 |
| 当前任务 | 当前正在执行的任务或最近取消的任务 | 当前任务状态 | 有任务后显示 | 受控终端不能只显示“已受管”，还要显示正在发生什么 |
| 本机信息入口 | 进入本机信息详情页 | 页面结构 | 始终可显示 | 用于核对机器名和硬件信息，并在需要时修正机器名 |
| 网络与服务器入口 | 进入网络与服务器详情页 | 页面结构 | 始终可显示 | 用于查看和修改 IP、DNS、网关、服务器地址，并做快速连通检测 |
| 本机桌面入口 | 进入本机桌面页 | 页面结构 | 始终可显示 | 用于当前终端的单机桌面准备、桌面编辑、默认启动设置和还原设置 |
| 运行状态 | CPU/GPU 使用率、内存已用/总量、磁盘已用/总量 | 本机实时数据 | 始终可显示 | 首页直接展示 4 个指标卡，超出阈值变色告警 |
| 服务器同步状态 | 终端与服务器的同步状态和最近同步时间 | 本机实时数据 | 始终可显示 | 外部连接卡片内展示 |
| 终端注册状态 | 当前终端是否已注册到服务器 | 本机实时数据 | 始终可显示 | 影响一键重置等功能的可用性 |
| 本机测试入口 | 进入本机测试页 | 页面结构 | 始终可显示 | 逐项测试网络连通性、硬盘读写速度和硬盘健康状态 |

补充说明：

1. TERM-01 里的“终端用途”只是结果展示，不是首页上的一个独立设置项。
2. 现场终端刚到教室时可能全部一样，甚至还没有安装云桌面管理系统；这时首页显示"未设定"是正常状态。
3. TERM-04 可以先用批量规则生成一版执行范围与写入方案，再在预览阶段逐台微调或排除。
4. 教师用途或其他特殊用途可以在后续单独部署中写入，不要求都在同一轮批量任务里完成。
5. TERM-01 允许本机做单终端容错修正；在首次装机、教师终端单机部署、晚补单机处理等场景里，也允许补齐当前本机的机器名、座位号、终端用途、网络配置、本机桌面、默认启动项和还原方式，但不在这里承担教室范围或批量写入。
6. TERM-04、TERM-05、TERM-07 里批量写入的机器名、座位号、终端用途、网络配置、服务器地址、桌面集合、默认启动项和还原方式，在单机场景下都应能在 TERM-01 的对应本机页逐项处理；区别只在作用范围，不在字段含义。

## 5. 首屏识别展示原则

TERM-01 首页不再把具体字段抽成统一摘要词，而是按下面的展示原则处理：

1. 只要空间允许，就同时展示机器名、当前 IP 地址、当前工作 MAC 和座位号。
2. 如果页面空间有限，至少保留"当前 IP 地址 + 当前工作 MAC"，机器名可退到次级位置。
3. 若当前已建立座位号，可把座位号作为补充上下文展示，但不拿它替代当前 IP 地址和当前工作 MAC 的定位作用。

导航栏身份信息格式已确认为：在线状态圆点 + "在线"/"离线"文字标签 + MAC 地址 + 机器名 + 座位号 + IP 地址，各项之间用视觉分隔符或背景色卡片隔开。

MAC 地址列表、硬盘 SN、更多网络细节进入“本机信息”或“网络与服务器”详情层，不挤在首屏。

## 6. 状态化页面稿

### 6.1 状态 A：首次进入管理系统

适用场景：终端刚完成底层安装，尚未建立教室关系、终端用途、服务器地址和桌面内容。

```text
┌──────────────────────────────────────────────────────────────────────────────┐
│ 可配置项           │ 外部连接           │ 硬件信息        │ 运行状态        │
│ 机器名：未命名     │ 服务器：未配置      │ 网卡：1 个      │ CPU  12%       │
│ 座位号：未分配     │ 状态：未配置        │ 处理器：i5      │ GPU   5%       │
│ 终端用途：未设定   │ 注册：未注册        │ 显卡：UHD 730   │ 内存 4/16GB    │
│ IP 地址：未配置    │ 同步：未同步        │ 内存：16GB      │ 磁盘 80/512GB  │
│ 默认启动：无桌面   │                     │ 硬盘：SN770     │                │
├──────────────────────────────────────────────────────────────────────────────┤
│ 单机功能                                                                     │
│ [本机配置] [服务器连接] [桌面管理] [一键替换] [一键重置] [本机测试]           │
│ 教室管理                                                                     │
│ [进入教室管理]                                                               │
└──────────────────────────────────────────────────────────────────────────────┘
```

说明：

1. 首页为四栏并排信息区（可配置项 / 外部连接 / 硬件信息 / 运行状态）+ 两组操作按钮。
2. 可配置项按信息→网络→桌面顺序：机器名、座位号、终端用途、IP 地址、默认启动桌面。不包含还原模式。
3. 硬件信息优先级：网卡列表、处理器、显卡、内存、硬盘。
4. 运行状态紧凑排列，与其他三栏等宽并排。
5. 整页在 1920×1080 分辨率下不出现页面滚动条。

### 6.3 状态 C：当前母机恢复后的系统直达规则

适用场景：软件退出后重新进入管理系统，发现本机仍作为当前母机。

1. 如果当前已有部署、维护或考试任务在执行，系统直接回到对应任务页。
2. 如果当前任务已经走到某个中间步骤，系统直接恢复到该任务步骤和当前进度，不额外插入中间过渡态。
3. 如果当前没有进行中的教室级任务，但当前母机对当前教室的管理关系仍有效，系统直接回到当前教室工作页。
4. 只有当这层管理关系已失效或当前任务上下文已不存在时，TERM-01 才回到普通首页状态。

### 6.4 状态 D：当前终端已受管，等待任务开始

适用场景：本机属于当前教室受控终端，母机已发起教室级任务，本轮已纳入执行，但本机尚未开始接收数据或指令。

```text
┌──────────────────────────────────────────────────────┐
│ 本机摘要                                             │
│ 当前工作MAC 08-7A-3C-11-2F-31                        │
│ IP地址 192.168.10.31 | 用途：学生机                   │
├──────────────────────────────────────────────────────┤
│ 本机状态                                             │
│ 等待任务开始                                         │
│ 当前教室：A401 机房 | 当前母机：192.168.10.21         │
│ 当前任务：单教室部署                                  │
├──────────────────────────────────────────────────────┤
│ 说明                                                 │
│ 本机已纳入本次执行，正在等待母机下发指令              │
├──────────────────────────────────────────────────────┤
│ [查看当前任务说明] [本机信息]                         │
└──────────────────────────────────────────────────────┘
```

说明：

1. 等待任务开始是受控终端进入任务后的第一画面，区别于"正在执行"。
2. 此状态下不允许进入桌面、修改网络或发起故障处理。
3. 如果等待时间过长，页面不自动退出，仍保持等待态直到母机下发指令或母机取消任务。

### 6.5 状态 E：当前终端已受管，本次正在执行

适用场景：本机属于当前教室受控终端，且本次已纳入执行，当前仍在执行中。

```text
│ 本机状态                                             │
│ 正在执行当前任务                                     │
│ 当前教室：A401 机房 | 当前母机：192.168.10.21         │
│ 当前任务：单教室部署 | 当前阶段：桌面传输 82%         │
├──────────────────────────────────────────────────────┤
│ 本机任务信息                                         │
│ 当前座位：A04 | 当前IP：192.168.10.31 | 计划配置IP：192.168.10.32 │
├──────────────────────────────────────────────────────┤
│ 可查看信息                                           │
│ [查看当前任务说明] [本机信息]                         │
└──────────────────────────────────────────────────────┘
```

说明：

1. 受控终端不会跳到 TERM-04；它仍停留在 TERM-01，只是切换成受控任务屏。
2. 母机看到的是整批任务进度，受控终端看到的是“本机当前任务进度”。
3. 如果当前终端属于教室受管集合但没有纳入本次执行，这里应显示“本次未纳入执行”，而不是伪装成掉线或失败。
4. 在这一状态里，不允许直接进入桌面、修改网络与服务器或发起故障处理。

### 6.6 状态 F：当前终端已受管，但本次未纳入执行

1. “本次未纳入执行”不是失败，也不是掉线。
2. 这类终端不展示传输百分比或失败原因。
3. 如果后续要单独处理，应由母机重新发起任务或进入其他专项流程。

5. 当前终端用途、所属教室、当前座位；其中终端用途和当前座位在首次装机或明确的单机准备场景下可编辑，其他场景只读回显。

## 10. 本机桌面页

本机桌面不是批量部署页的缩小版，而是当前终端的单机桌面入口。它的目标是让教师终端、晚补的一台学生终端，或其他明确的单机场景，都能在不进入教室级批量任务的前提下完成桌面与启动/还原设置处理。

### 10.1 页面职责

1. 展示当前终端已有桌面列表和磁盘空间占比。
2. 允许对每个已有桌面执行：进入桌面编辑、复制桌面、设为默认启动、设置还原模式、隐藏/显示、删除。
3. 允许导入桌面：支持基础镜像文件（.vhd/.vhdx/.img/.iso/.wim）和成品桌面包（.deskpkg）两种导入路径。基础镜像导入时需经过创建桌面流程（选择数据盘、选择是否物理部署），成品桌面包导入时直接生成桌面跳过创建流程。
4. 允许导出桌面为自定义后缀的桌面包（.deskpkg），用于跨终端离线传递（如装在 U 盘带到另一间未安装的教室导入）。
5. 展示每个桌面的上传状态（是否已上传到服务器），让用户知道未上传的桌面删除后不可恢复。
6. 展示每个桌面是否为物理部署（物理部署的桌面使用独立物理分区而非虚拟磁盘）。
7. 在教师终端单机部署、晚补单机处理等场景里，用来补齐本机桌面、默认启动和还原方式。

### 10.2 磁盘空间摘要

页面顶部展示磁盘空间占比摘要条，使用 SVG 饼图 + 文字数值共同呈现：

1. 桌面占用空间。
2. 系统占用空间。
3. 剩余可用空间。
4. 总容量和估算可新建桌面数。

目的是让用户自行估算还能创建或复制多少桌面，不同区段使用不同颜色区分。

### 10.3 桌面操作区

1. [导入桌面]：点击后先选择导入类型（基础镜像 or 成品桌面包），再通过操作系统文件选择框选择文件。操作系统文件选择框使用原生实现。
2. [导出桌面]：将当前终端的桌面导出为 .deskpkg 桌面包。

### 10.4 桌面卡片列表

每个已有桌面以独立卡片形式展示。每张桌面卡片包含以下信息和操作：

**卡片信息：**

1. 桌面名称（安装工程师手动输入，不存在自动版本号）。
2. 默认启动标记：如果该桌面是当前默认启动项，显示标记标签。
3. 可见性标记：使用"显示"和"已隐藏"中文标签。
4. 上传状态：使用"已上传"和"未上传"标签，让用户知道未上传到服务器的桌面删除后无法恢复。
5. 物理部署标记：如果该桌面使用物理分区部署，显示"物理部署"标签。
6. 操作系统类型。
7. 基础镜像名称（终端侧只需展示从哪个镜像派生，不展示完整快照树）。
8. 最近更新时间。
9. 备注（每次编辑桌面时可写入备注，完成后在终端和平台都可查看）。
10. 数据盘列表：每个数据盘展示盘符/挂载路径、大小，是否被其他桌面共享。

**卡片内联设置：**

1. 还原模式：每个桌面独立设置还原模式（还原系统盘保留数据盘 / 还原系统盘和数据盘 / 不还原），支持直接修改。

**卡片操作按钮：**

1. [进入桌面编辑]：重启终端进入该桌面环境进行编辑，编辑完成时弹出备注输入框。
2. [复制桌面]：基于该桌面复制一个新桌面到当前终端。
3. [设为默认]：将该桌面设为默认启动项。
4. [隐藏] / [显示]：切换该桌面在启动菜单中的可见性。
5. [删除]：删除该桌面。删除默认启动桌面时，系统自动将默认顺延到下一个可用桌面。删除桌面时，如果该桌面的数据盘还被其他桌面引用则保留数据盘。

### 10.5 导入桌面流程

导入桌面区分两条路径：

**路径 A：基础镜像文件**

选择文件后进入创建桌面流程，需要配置：

1. 桌面名称。
2. 数据盘：大小、盘符/挂载路径（默认 D:，第二个顺延为 E:）。可以选择复用当前终端已有的数据盘，也可以新建隔离的数据盘。
3. 部署方式：虚拟部署（VHD）或物理部署（独立物理分区）。物理部署不能挂载虚拟磁盘，只能使用真实物理磁盘的隔离分区。
4. 还原模式。
5. 备注。

**路径 B：成品桌面包（.deskpkg）**

选择文件后直接生成桌面，跳过创建流程。桌面包内已包含完整配置信息。

### 10.6 桌面上传机制

1. 终端在空闲时刻自动与服务器确认桌面文件状态。
2. 如果本地桌面文件在服务器不存在相同哈希的文件，终端需要上传；服务器在收到确认请求时与一台终端建立传输连接，其他拥有同一哈希文件的终端被告知无需传输。
3. 桌面上传与终端状态/配置的上报不同：状态和配置是轻量数据（在线状态、CPU 使用率、已部署桌面列表、启动项设置等），桌面文件是大文件。
4. 未上传到服务器的桌面如果被用户删除，将无法恢复，页面必须显式提醒。

### 10.7 桌面数据结构说明

桌面实际包含三层结构：原始镜像 → 第一层持久快照 → 第二层临时快照。

1. 用户每次进入桌面编辑进入的是临时快照。
2. 还原模式下，每次删除临时快照并重新构建。
3. 编辑合并模式下，将临时快照合并到持久快照层，再重建新的临时快照。
4. 终端侧只需要展示桌面从哪个基础镜像派生（根节点名称），其他信息由安装工程师通过备注或桌面名称提供。
5. 平台侧需要展示完整快照树结构：每级快照的文件名、大小、更新时间。

### 10.8 数据盘说明

1. 当前数据盘在桌面之间没有共用的默认行为，每次建桌面默认添加的数据盘是新的、互相隔离的。
2. 建桌面时如果需要添加数据盘，可以选择当前终端已有的数据盘（复用），也可以新建。
3. 盘符或挂载路径需要提供默认值：第一个数据盘默认 D:，第二个顺延为 E:（Windows）或对应挂载路径（Linux）。
4. 删除桌面时，如果该桌面的数据盘还被其他桌面引用，则保留数据盘。
5. 数据盘是否被其他桌面使用，应像桌面卡片一样在查看时让用户知道。
6. 物理部署的桌面不能挂载虚拟磁盘，只能使用真实物理磁盘的隔离分区。一个桌面可以使用多个数据盘。

### 10.10 平台侧桌面信息差异

桌面相关概念在终端侧和平台侧必须保持一致。如果终端侧有的信息，平台侧也应该有。两侧的差异仅在于：

1. 平台侧教室管理的终端详情页应展示：该终端部署了哪些桌面、每个桌面的设置（包括默认启动、还原模式、可见性）。这属于终端配置的一部分。
2. 平台侧全平台桌面资产页不应出现"默认启动"或"显示/隐藏"字样——默认启动和可见性是终端级设置，每个终端可能不同。
3. 平台侧桌面详情展示完整的镜像快照文件链（终端侧只展示基础镜像名称），包括每级快照的文件名、大小、更新时间。
4. 平台侧教室级桌面信息总览应展示每个桌面被多少终端使用。
5. 不存在自动标签（如教学、原始、考试），也没有自动版本号（v1/v2/v3）；如果需要类似信息，由安装工程师在桌面名称或备注中手动输入。

### 10.11 边界说明

1. 这里处理的是当前这台终端，不处理教室级批量范围。
2. 当前终端未受控时，允许直接改单机桌面。
3. 当前母机进入单教室部署时，本机桌面准备改在 TERM-04 的起始阶段直接完成。
4. 当前终端已受控且任务正在执行时，只允许查看，不允许改写。
5. 用户完成桌面处理并再次重启后，回到发起页面。
6. [设为默认] 只改当前终端后续启动行为，不触发教室级写入。
7. 删除默认启动桌面时自动顺延到下一个可用桌面。
8. 危险操作使用自定义确认弹窗，删除未上传桌面时需额外提醒不可恢复。
9. 导入桌面使用操作系统原生文件选择框。
10. 桌面上传是终端空闲时自动进行的，不需要用户手动触发。

## 11. 本机测试页

本机测试页用于在现场逐项检测终端的网络和硬件状态。由安装工程师主动触发，不是后台自动运行。

### 11.1 页面职责

1. 逐项测试本机网络与硬件状态，提供直观的通过/失败判断。
2. 每个测试项支持单独重新测试（重测按钮），不需要整体重跑。
3. 不包含"硬件基线检测"——只做实际状态的通过/失败判断。

### 11.2 测试项目（按页面顺序）

**第一项：网络连通性测试**

1. 本机 IP 地址。
2. 网关地址及可达性。
3. 服务器地址及可达性。
4. [重新测试] 按钮。

**第二项：硬盘读写速度测试**

1. 硬盘型号和序列号。
2. 顺序读取速度（Seq Q32T1）。
3. 随机读取速度（4K Q32T1）。
4. 顺序写入速度（Seq Q32T1）。
5. 随机写入速度（4K Q32T1）。
6. [重新测试] 按钮。

**第三项：硬盘健康状态测试**

1. SMART 状态（正常/异常）。
2. 温度。
3. 剩余寿命估算。
4. [重新测试] 按钮。

### 11.3 服务器远程测试扩展

在服务器侧远程触发终端自检时，除以上三项外额外增加广播风暴测试。终端本机测试页不包含广播风暴测试（该测试需要服务器协调多台终端同时执行）。

## 12. 首页运行状态区

### 12.1 区域职责

在首页直接展示终端当前运行状态，供安装工程师快速判断终端资源使用情况。不需要进入子页面。

### 12.2 展示内容

首页"运行状态"卡片内，以 4 个指标卡并排展示：

1. **CPU 使用率**：百分比数值，超过 80% 红色，超过 60% 黄色，否则绿色。
2. **GPU 使用率**：百分比数值，配色规则同 CPU。
3. **内存**：已用/总量（GB），同时显示百分比；超过 85% 红色，超过 70% 黄色。
4. **磁盘**：已用/总量（GB），同时显示百分比；配色规则同内存。

### 12.3 首页外部连接卡片

在首页"外部连接"卡片内展示：

1. 服务器地址。
2. 服务器连接状态（已连接/未配置）。
3. 终端注册状态（已注册/未注册）。
4. 最近同步时间。

### 12.4 数据来源

终端运行状态数据来自本机实时采集，通过 SSE 定期推送到前端。原型中由服务器端 `randomize-metrics` 动作模拟随机变化。
