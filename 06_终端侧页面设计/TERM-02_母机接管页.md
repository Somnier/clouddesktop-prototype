## 1. 这页真正要解决什么问题

TERM-02 只解决“接管哪间教室、纳管哪些终端、能不能继续接管”这件事，不负责完成部署规划。

它必须回答：

1. 当前是初次接管，还是接管已有教室。
2. 本次扫描发现了哪些可管理终端。
3. 这些终端里哪些属于同一教室主组，哪些未绑定教室，哪些属于其他教室组。
4. 本次接管能否继续，以及风险在哪里。

TERM-02 不回答：

1. 这批终端最终要写成什么 IP。
2. 这批终端最终要写成什么机器名。
3. 座位号如何规划。
4. 桌面如何制作和分发。

这些内容属于后续部署或维护流程。

## 2. 这页与其他页面的关系

1. TERM-02 从 TERM-01 的“进入教室管理模式”进入。
2. TERM-02 接管成功后进入 [TERM-03_母机工作页.md](./TERM-03_母机工作页.md)。
3. TERM-02 结束时只形成“当前教室范围”和“当前接管结果”，不形成部署参数。
4. 本地教室成员基线摘要在这里仅用于判断“是不是这间教室的成员”，不扩展为部署规划或服务器创建教室的数据源。

## 3. 当前页依赖的数据层

### 3.1 母机本机当前信息

1. 本机机器名。
2. 本机当前 IP 地址。
3. 本机当前工作 MAC。
4. 本机是否已经进入教室管理模式。

### 3.2 本次扫描结果数据

1. 本次扫描发现终端数。
2. 每台已响应终端的扫描接入标识。
3. 本次扫描归组数。
4. 本次扫描未绑定数。
5. 是否存在其他教室组。

### 3.3 终端本地教室绑定摘要

1. 当前所属教室名称。
2. 首次建教时间。
3. 最近一次写入时间。
4. 最近一次写入成员数。
5. 教室成员基线摘要。

### 3.4 当前页不能直接知道的事情

1. 某台这次没响应的终端到底属于本教室还是不属于本教室。
2. 当前未绑定终端到底是新终端，还是旧终端曾漏写教室绑定。
3. 这些终端最终要写入什么 IP、计算机名和座位号。

## 4. 页面字段定义

| 字段 | 含义 | 来源 | 何时可显示 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| 本次扫描发现终端数 | 本次扫描中已响应的可管理终端数量 | 本次扫描结果数据 | 扫描完成后 | 只说明这次发现了多少台，不等于在线数 |
| 扫描接入标识 | 这台终端在本次扫描里可用于区分自己的接入信息 | 本次扫描结果数据 | 扫描到单台终端后 | 优先使用当前工作 MAC，临时接入地址只能作为辅助信息 |
| 最近一次写入成员数 | 某教室绑定结果上次成功写入时记录的成员总数 | 终端本地教室绑定摘要 | 识别到已有教室后 | 是历史规模，不是当前在线规模 |
| 本次扫描归组数 | 本次扫描里返回同一教室绑定结果的终端数量 | 本次扫描结果数据 + 本地绑定摘要 | 识别到已有教室后 | 是本次归组结果 |
| 本次扫描未绑定数 | 本次扫描里已响应但未返回教室绑定信息的终端数量 | 本次扫描结果数据 | 出现主组 + 未绑定终端时 | 只说明未绑定，不说明成因 |
| 其他教室组数量 | 本次扫描里识别到的其他不同教室组数量 | 本次扫描结果数据 + 本地绑定摘要 | 出现多组教室时 | 表示网络内存在其他教室组 |
| 占用状态 | 当前目标教室是否仍由其他母机管理 | 接管状态信息 | 进入接管确认或占用校验时 | 只回答当前能否继续接管 |

## 5. 终端行识别字段如何显示

TERM-02 的终端行不能默认使用最终 IP、最终机器名或最终座位号。

接管页更稳妥的展示原则是：

1. 始终展示当前工作 MAC。
2. 如果扫描框架还能提供临时接入信息，就与当前工作 MAC 一起展示。
3. 如果终端已有用途标记，可以作为辅助信息补充，但不伪装成稳定身份。
4. 只要扫描结果或未绑定终端列表需要人工勾选，就应支持排序和分页；可排序字段、默认排序和分页策略按当前结果规模和识别重点确定，不写死。

补充规则：

1. 如果后续已有稳定教室组，首页显示重点应是“教室组卡片”，而不是把每台终端都假装成已有稳定身份。
2. 最终 IP、机器名、座位号属于 [TERM-04_单教室部署流程.md](./TERM-04_单教室部署流程.md) 的规划内容，不应在 TERM-02 提前写成既定事实。
3. 教室成员基线摘要的使用范围只限“判断当前是否属于这间教室”，不扩展到后续日常运维。

├──────────────────────────────────────────────┤

│ 教室名称输入框：未命名教室-20260326-142530   │
│ 提示：建议填写确切教室名，后续导出时默认带出 │
├──────────────────────────────────────────────┤

3. 默认教室名应按当前时间自动生成，避免重复；正式导出前仍建议改成确切教室名。

## 7. 关键弹层与确认动作

### 7.1 并入所选终端确认弹层

在“主组明确，但存在未绑定终端”状态下点击 [并入所选终端] 时，应增加确认弹层：

```text
┌──────────────────────────────────────────────────────┐
│ 确认并入所选终端                                     │
├──────────────────────────────────────────────────────┤
│ 将把 2 台所选终端并入 A401 机房                      │
│ 这些终端会写入当前教室名称、建教时间和成员基线摘要  │
│ 这里不写 IP、座位号、机器名或网络配置               │
├──────────────────────────────────────────────────────┤
│ [确认并入] [返回检查]                                │
└──────────────────────────────────────────────────────┘
```

### 7.2 多教室风险继续确认弹层

在“网络中存在其他教室”状态下点击 [继续接管所选组] 时，应增加二次确认：

```text
┌──────────────────────────────────────────────────────┐
│ 确认继续接管所选组                                   │
├──────────────────────────────────────────────────────┤
│ 当前网络内仍存在其他教室组                           │
│ 后续批量操作仅作用于你当前选中的这一组               │
│ 其他组不会被纳入本次操作                             │
├──────────────────────────────────────────────────────┤
│ [确认继续] [返回重新选择]                            │
└──────────────────────────────────────────────────────┘
```

### 7.3 扫描偏少继续确认弹层

在“扫描结果偏少提醒”状态下点击 [继续当前结果] 时，应增加确认弹层：

```text
┌──────────────────────────────────────────────────────┐
│ 确认按当前结果继续                                   │
├──────────────────────────────────────────────────────┤
│ 最近一次写入成员数 48，本次仅归组 36                  │
│ 按当前结果继续后，本次教室范围将只按已发现结果建立    │
├──────────────────────────────────────────────────────┤
│ [仍继续] [返回重新扫描]                              │
└──────────────────────────────────────────────────────┘
```

### 7.4 取消接管确认弹层

在扫描完成或已识别出主组后点击 [取消] 或 [返回首页] 时，应增加确认弹层：

```text
┌──────────────────────────────────────────────────────┐
│ 确认放弃本次接管                                     │
├──────────────────────────────────────────────────────┤
│ 当前扫描结果不会写入为教室接管结果                   │
│ 本机将返回普通首页状态                               │
├──────────────────────────────────────────────────────┤
│ [确认返回] [继续当前接管]                            │
└──────────────────────────────────────────────────────┘
```

## 8. 页面结论

1. TERM-02 的职责到“识别和接管教室”为止。
2. TERM-02 不提前吞掉部署规划、IP 规划、座位规划这些后续流程。
3. TERM-02 可以提示风险，但不能用不完整数据假装知道更深层原因。
4. 默认教室名称应采用“未命名教室-日期-序号”这类唯一格式，不伪装成真实教室名。
