## 1. 这页真正要解决什么问题

TERM-07 是考试场景流程页，但它不是另一套独立桌面系统。

这页真正要解决的是三件事：

1. 先从当前母机已有本机桌面里选择这次考试要下发的一个或多个桌面。
2. 再从这批桌面里选定考试默认启动桌面，并把桌面集合和考试启动设置批量应用到这轮勾选终端。
3. 考后再把这轮终端恢复到考前的日常启动设置。

因此 TERM-07 也必须守住和 TERM-04、TERM-05 一样的边界：

1. 当前教室受管集合，是这间教室当前被母机管理的终端总集合。
2. 这轮勾选终端，是这次考试设置真正要处理的终端集合。
3. 保持现状终端，是留在教室里但这轮不动的终端。
4. 本次考试要下发的桌面集合可以是当前母机已有的一个或多个本机桌面。
5. 当前母机需要调整考试桌面时，应先处理本机桌面，再进入 TERM-07。
6. 真正批量执行的，是考试桌面下发和考试启动设置，而不是一批桌面对象。

## 2. 这页与其他页面的关系

1. TERM-07 从 [TERM-03_母机工作页.md](./TERM-03_母机工作页.md) 的考试场景入口进入。
2. 如果需要准备考试桌面，TERM-07 应在桌面选择步骤提供单独的 [修改本机桌面] 按钮。点击后终端重启进入当前母机的本机桌面处理；在那里可新建桌面、复制已有桌面，或进入桌面编辑；完成这次桌面处理并再次重启后，系统回到 TERM-07 当前步骤。TERM-07 自身不再把桌面制作动作并进批量选择层。
3. TERM-07 与 [TERM-05_教室维护流程.md](./TERM-05_教室维护流程.md) 共享“多桌面下发集合 + 批量终端执行”的纯批量边界。
4. TERM-07 执行期间，受控终端仍停留在 [TERM-01_管理系统首页.md](./TERM-01_管理系统首页.md) 的受控任务屏，只表达本机当前进度。
5. 考试前设置完成、考后恢复完成后，结果都应回写 TERM-03 的任务摘要和终端状态。
6. 如果考试过程中只剩个别终端需要脱离整批单独处理，应转交 [TERM-06_故障处理流程.md](./TERM-06_故障处理流程.md) 的单机排障视角。
7. 这轮不参与的终端仍属于当前教室受管集合；后续如需单机处理，再进入对应本机入口或故障处理流程，TERM-07 本身不展开后续补路说明。

## 3. 当前页依赖的数据层

### 3.1 当前教室上下文

1. 当前教室名称。
2. 当前母机身份。
3. 当前学生终端数。
4. 当前教师终端数。
5. 当前受管终端集合。
6. 最近刷新时间。

### 3.2 当前母机考试桌面选择状态

1. 当前母机已有桌面列表，至少展示桌面名称、时间和备注。
2. 当前勾选的考试桌面下发集合。
3. 当前考试桌面的数据盘方案。
4. 考试默认启动桌面。

### 3.3 考试设置范围与当前值

1. 当前教室受管集合。
2. 这轮勾选终端。
3. 保持现状终端。
4. 这轮是否存在保持现状终端。
5. 当前默认启动目标。
6. 这次要写入的考试默认启动目标。
7. 其他桌面入口是否隐藏。
8. 还原模式。
9. 考前当前启动设置快照。

### 3.4 执行与恢复状态

1. 桌面下发进度。
2. 考试启动设置进度。
3. 当前考试设置结果。
4. 考前快照保存状态。
5. 考后恢复进度。

## 4. 页面字段定义

| 字段 | 含义 | 来源 | 何时可显示 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| 当前桌面列表 | 当前母机已有并可直接用于考试的本机桌面集合 | 当前母机考试桌面选择状态 | 进入后 | 允许多选下发 |
| 考试桌面集合 | 这次考试实际要下发的一组桌面 | 当前母机考试桌面选择状态 | 选中后 | 允许多选 |
| 考试默认启动桌面 | 这次考试实际要作为默认启动项的那一个桌面 | 考试设置范围与当前值 | 确认前 | 必须从考试桌面集合里选择 |
| 这轮勾选终端 | 本次考试设置真正要处理的终端集合 | 考试设置范围与当前值 | 勾选后 | 与受管集合分开表达 |
| 保持现状终端 | 当前教室里这轮不动的终端集合 | 考试设置范围与当前值 | 勾选后 | 不参与本次考试设置 |
| 保持现状终端说明 | 这轮不执行考试设置的终端说明 | 考试设置范围与当前值 | 勾选后 | 需要同时说明本次不修改和沿用的启动设置 |
| 其他桌面入口显示 | 其他桌面入口是否隐藏 | 考试设置范围与当前值 | 确认前 | 这是考试场景核心控制项之一 |
| 考前当前启动设置 | 开始考试设置前读取到的默认启动目标和入口显示结果 | 考试设置范围与当前值 | 执行前 | 供考后一键恢复 |
| 考后恢复目标 | 考后要恢复到什么日常启动设置 | 执行与恢复状态 | 恢复前 | 恢复的是考前快照，不是抽象类别 |

## 5. 流程骨架

### 5.0 考试场景的前提按考试动作本身判断

1. TERM-07 要求当前母机仍处于可管理当前教室的状态。
2. 如果当前还没有合适的考试桌面，或某个考试桌面内容还要继续准备，可在桌面选择步骤点击 [修改本机桌面]；在那里可新建桌面、复制已有桌面，或进入桌面编辑；完成这次桌面处理并再次重启后，系统回到 TERM-07 当前步骤。
3. 因此考试场景依赖考试桌面准备，但 TERM-07 的批量主线只负责选择已有桌面和批量写考试设置。

### 5.1 先选择已有的考试桌面，再批量写考试设置

1. 先在当前母机已有桌面列表中勾选 1 个或多个考试相关桌面。
2. 如果需要安装考试软件或做考试环境校验，应在当前步骤点击 [修改本机桌面]，进入当前母机的本机桌面处理；在那里可新建桌面、复制已有桌面，或进入桌面编辑；完成这次桌面处理并再次重启后，系统回到 TERM-07 当前步骤。
3. 选好考试桌面后，直接进入范围确认和考试设置确认。
4. 只要桌面列表需要人工勾选，就应支持排序和分页；可排序字段和分页策略按当前桌面规模与考试场景决定，不写死。

### 5.2 再确认这轮勾选终端和考试启动设置

1. 明确这轮勾选终端和保持现状终端。
2. 如果教室里存在教师终端，也只是在同一终端列表里选择参与或保持现状，不单独拆额外判断页。
3. 系统在真正写入考试设置前，应先读取并保存考前当前启动设置快照。
4. 确认页必须同时展示考试桌面集合、考试默认启动桌面、这轮勾选终端、保持现状终端、其他入口显示和还原模式。
5. 保持现状终端不静默消失；确认页要明确写出本次不修改，并回显沿用的启动设置。
6. 只要终端列表需要人工勾选，就应支持字段排序、分页和已勾选终端拖动排序；可排序字段、默认排序字段和分页策略按当前教室规模决定，不写死。
7. 如果某些终端最终保持现状，结果页继续按“本次不修改”显示，不把它们并进失败终端。

### 5.3 考试前设置和考后恢复都要沿用同一批量入口

1. 考试前设置以批量任务执行。
2. 当前教室本地完成，即表示这轮考试准备已对这批终端生效；服务器侧同步不阻塞当前教室主线。
3. 考后恢复时，沿用同一批量入口，把这轮终端恢复到考前快照记录的默认启动和入口显示状态。
4. 考试相关批量任务执行中必须支持主动取消；取消后已完成终端保持当前结果，未完成终端不按成功处理。

## 8. 页面结论

1. TERM-07 不是另一套独立桌面系统，而是“选已有考试桌面 + 批量写考试设置 + 考后批量恢复”的考试场景页。
2. 考试桌面可以多选成为考试桌面集合，但来源只能是当前母机已经准备好的本机桌面。
3. 考试场景沿用 TERM-04 和 TERM-05 的纯批量骨架：桌面选择步骤提供 [修改本机桌面] 按钮，供当前母机补改考试桌面；批量步骤本身只负责选已有桌面、确认范围和考试设置，然后批量执行。
4. 某些终端如保持现状，应直接写清本次不修改和沿用的启动设置，不单独展开教师终端补路说明。
5. 考后恢复恢复的是考前快照记录的启动设置，而不是空泛的“恢复普通模式”。
