# WEB-05 系统运维

> 页面角色：服务器级低频运维操作的独立入口。
> 执行主体：系统管理员（全局）。
> 数据来源：服务器本体状态（license、证书、IP 地址），平台账号体系。

## 1. 页面职责

集中承接授权管理、服务器地址变更和校区账号管理三类低频但流程独立的运维操作。日常查看类指标属于 WEB-01 平台总览，不在本页重复。

## 2. 页面结构

```text
┌──────────────────────────────────────────────────────────────┐
│ 系统运维                                                      │
├────────────────────┬─────────────────────────────────────────┤
│ ▸ 授权管理         │                                         │
│ ▸ 服务器地址变更   │    ← 当前选中区块的操作面板 →            │
│ ▸ 校区与账号管理   │                                         │
├────────────────────┴─────────────────────────────────────────┤
│                    操作面板区                                  │
└──────────────────────────────────────────────────────────────┘
```

页面分左导航三个区块，右侧为当前区块的操作面板。

### 3.2 字段说明

| 字段 | 说明 |
| :--- | :--- |
| 授权容量 | 当前授权许可的最大终端数 |

- **导入授权文件**：选择文件后上传，成功后刷新容量和状态。

### 3.4 空态

- 未导入任何授权：显示"尚未导入授权文件"和导入按钮。

## 4. 服务器地址变更

### 4.1 操作背景

> 参考：[云桌面终端与服务器按需同步说明.md](../05_云桌面事实/云桌面终端与服务器按需同步说明.md) §4.2 第 6–8 条。
> 参考：PRD FR-22 服务器地址变更。

服务器地址变更是低频但流程复杂的独立操作。终端侧改的是业务配置中的服务器地址，不向终端下发证书；若启用 HTTPS，新 SSL 证书在本页面单独导入。

```text
│                                                              │

```

### 4.3 流程说明

**第一步：通知在线终端**

1. 输入新服务器地址后点击"通知在线终端"。
2. 服务器向当前在线终端推送新地址配置。
3. 通知完成后显示送达结果：已送达台数、未送达台数、离线终端台数。
4. 离线终端不在这一步通知范围内。

**第二步：确认物理 IP 变更已完成**

1. 管理员在服务器物理网络侧完成 IP 地址变更。
2. 点击确认按钮，平台记录地址变更生效。

**第三步：查看未送达终端清单**

1. 生成未送达终端清单，包含未送达和离线终端。
2. 清单字段：教室、座位号、机器名、IP、后续处理入口。
3. 默认后续处理入口为目标终端本机的 TERM-01 [网络与服务器]；只有该终端已经进入故障处理意图时才转 TERM-06。
4. 支持导出未送达终端清单。

### 4.4 SSL 证书管理

- 当前证书状态：有效 / 即将过期 / 已过期。
- 导入新证书：选择证书文件上传。
- 终端不接收证书下发；证书由服务器侧在此处单独导入。

### 4.5 空态

- 从未变更过服务器地址：只显示当前地址和输入框，无历史清单。

### 5.2 账号体系说明

> 参考：PRD §2.1–2.3 人员与账号模型。

- **线下角色**（安装工程师、机房管理员、技服工程师、教师）与**平台账号**（系统管理员、校区子管理员）分开建模。
- 全局管理员（系统管理员）可管理所有校区。
- 校区子管理员只能查看和操作本校区数据。
- 当前阶段只支持密码修改；更细粒度的权限管理和账号生命周期属于二期。

### 5.3 空态

- 无校区时：显示"尚未创建校区"和新增入口。
