# WEB-04 告警中心

> 页面角色：校区级告警汇总与分发入口。
> 执行主体：系统管理员、校区子管理员或机房管理员（远程查看）。
> 数据来源：终端上报的告警记录和服务器自生成的系统告警。

## 1. 页面职责

集中展示当前校区所有活跃告警，支持按严重度和时间排序，并提供跳转到具体教室或终端的入口。

## 3. 每条告警的字段

| 字段 | 说明 |
| :--- | :--- |
| 严重度标签 | 高（红色）/ 中（黄色）/ 低（灰色） |
| 标题 | 告警摘要 |
| 详情 | 告警详细描述 |
| 来源 | 教室名称（可点击跳转 WEB-02 教室详情告警 Tab）+ 终端信息（座位号、机器名、IP，可点击跳转 WEB-02A 终端详情） |
| 时间 | 相对时间 |

## 4. 筛选与排序

> 参考：PRD FR-14 第 6–7 条。

### 4.1 筛选

- **按教室**：下拉选择特定教室，只显示该教室的告警。
- **按类型**：下拉选择告警类型（硬件不一致、外设未连接、硬盘异常、终端离线、服务器资源异常等）。
- 筛选条件可组合使用。

### 4.2 排序

- **按严重度**（默认）：高 → 中 → 低，同级按时间倒序。
- **按时间**：最新在前。
- 点击排序按钮可在升序和降序之间切换。

## 5. 与 WEB-02 教室告警 Tab 的关系

- WEB-02 教室详情的告警 Tab 只展示单教室的告警。
- WEB-04 汇总当前校区所有教室的告警，额外显示教室来源列，方便跨教室定位。
- 两处的告警条目格式一致。

## 6. 空态

- 无活跃告警时：显示"当前无活跃告警"。
