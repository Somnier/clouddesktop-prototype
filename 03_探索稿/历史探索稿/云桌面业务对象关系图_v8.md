# 云桌面业务对象关系图

> **设计思想**：
> 1. **拒绝 AI 式的过度抽象**：坚决摒弃“信息采撷”、“环境注入”等生造的总结性黑话，回归人类产品经理日常沟通的自然语言（如“导出”、“读取”、“安装U盘”），让人一看就懂。
> 2. **孤岛间的业务探讨价值**：在终端底层域引入了`本地告警记录`。它与服务器的告警池看似重叠，实则是为了支撑“脱机断网状态下终端依然能发现硬件被拔插”的业务场景。图上画出这两个独立的实体，天然就能在会议中引发“这两端告警如何同步、由谁处理”的业务探讨。
> 3. **工具生命周期的重定位**：将“安装U盘”从“一次性交付域”改为了更宽泛的“辅助工具域”。因为它不仅用于初始安装，在后续加装软件、部署纯净考试环境时依然会被拔插使用，它是一个贯穿业务周期的实体工具。

```mermaid
classDiagram
    %% ================= 服务器域 =================
    namespace 服务器域 {
        class 服务器 {
            IP地址
            域名
            CPU与内存使用率
        }
        class 授权许可证书 {
            可用授权点数
            有效期限
        }
        class 终端云档案 {
            已登记的MAC地址
            绑定的IP与计算机名
            所属教室与座位
            实时在线状态
        }
        class 服务器告警记录 {
            告警类型
            异常源设备(MAC)
            溯源位置(教室与座位号)
            处理状态
        }
        class 云端基础镜像 {
            OS类型
        }
        class 云端快照 {
            桌面名称
            增量差异数据
        }
    }

    %% ================= 辅助工具域 =================
    namespace 辅助工具域 {
        class 安装U盘 {
            预装底层系统
            携带基础镜像
        }
        class 教室终端清单 {
            单教室全体终端数据(Excel格式)
        }
    }

    %% ================= 终端底层域 =================
    namespace 终端底层域 {
        class 物理终端 {
            物理MAC地址
            当前工作MAC地址
            硬盘SN码
            硬件规格
            外设连接状态
        }
        class 终端配置 {
            分配的IP地址
            计算机名
            所属教室与座位号
        }
        class 终端运行状态 {
            网络联机状态
            CPU与内存负载
        }
        class 本地告警记录 {
            告警类型
            处理状态
        }
        class 本地基础镜像 {
            OS类型
        }
        class 本地快照 {
            桌面名称
            还原模式
        }
    }

    %% ================= 终端引导域 =================
    namespace 终端引导域 {
        class 启动菜单 {
            可见启动项列表
            默认启动项
        }
    }

    %% ================= 终端桌面域 =================
    namespace 终端桌面域 {
        class 桌面 {
            正在运行的操作系统环境
        }
        class 云桌面客户端 {
            后台自动上传桌面()
            接收关机与重启指令()
        }
        class 教室管控软件 {
            一键开机(WOL)()
            一键关机()
        }
    }

    %% ================= 业务连线 (自然业务语言描述) =================

    %% 工具与终端的交互
    安装U盘 ..> 物理终端 : 为裸机安装系统与镜像 (日常加装与考试部署也可用)
    物理终端 ..> 教室终端清单 : 部署完成后导出清单数据
    教室终端清单 ..> 终端云档案 : 导入服务器完成终端注册
    终端云档案 --> 物理终端 : 云端档案唯一对应物理设备

    %% 终端基础属性与告警
    物理终端 *-- 终端配置
    物理终端 *-- 终端运行状态
    物理终端 *-- 本地告警记录 : 离线时也可记录硬件变更

    本地告警记录 <..> 服务器告警记录 : 联机时同步告警与处理状态
    本地基础镜像 ..> 云端基础镜像 : 首次联机上传备份
    本地快照 <..> 云端快照 : 桌面增量数据的同步

    %% 启动与运行
    终端配置 ..> 桌面 : 桌面启动时读取配置以获取IP和计算机名
    启动菜单 --> 本地快照 : 决定开机进入哪个桌面
    本地快照 --> 桌面 : 加载为运行环境

    桌面 *-- 云桌面客户端 : 后台驻留
    桌面 o-- 教室管控软件 : 教师机专有
    教室管控软件 ..> 物理终端 : 局域网唤醒其他终端
    教室管控软件 ..> 云桌面客户端 : 发送关机等控制指令
```
