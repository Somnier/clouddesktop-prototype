# 云桌面业务对象关系图 v7 (PM 深度推演版)

> **版本说明 (v7)**：
> 这一次，我们不再是为了“精简”而精简，而是为了“精准表达业务逻辑”而重构。
> 1. **独立出「实施交付域」**：深刻理解了“安装U盘”与“终端”的生命周期差异。U盘和《教室终端清单》本质上是脱机实施阶段的“摆渡工具”和“交付产物”，它们不应永远挂载在终端基座上，因此为其单独划域。
> 2. **实体拆分的“为什么”**：
>    - 终端的「逻辑身份配置」与「运行状态」必须拆开。因为配置（IP/计算机名）是需要被上层 Guest OS（桌面）读取注入的，而状态（CPU负载）只是为了上报。它们的业务消费方和数据流转方向完全不同。
>    - 服务器的「节点配置」与「运行状态」合并。因为在产品早期规划中，服务器作为中心节点，其自身的软硬件拆分并不驱动跨域的核心业务逻辑流转，过度拆分反而增加认知负担。
> 3. **淡化“服务”，聚焦“能力”**：早期不急于定性“XX服务”部署在哪，而是看实体是否具有某种业务能力（如桌面里的驻留代理端负责同步和听令）。

```mermaid
classDiagram
    %% ================= 服务器域 =================
    namespace 服务器域 {
        class 服务器节点 {
            网络标识(IP/域名)
            平台运行负载
        }
        class 授权凭证 {
            可用点数
        }
        class 终端云档案 {
            已登记网络与硬件信息
            所属教室与座位
            实时在线状态
        }
        class 统一告警池 {
            异常分类与溯源
        }
        class 云端资产库 {
            服务器端基础镜像
            服务器端快照
        }
    }

    %% ================= 实施交付域 =================
    namespace 实施交付域 {
        class 便携部署介质 {
            预装底层系统
            携带基础镜像
        }
        class 教室建档清单 {
            单教室实体全量快照(Excel)
        }
    }

    %% ================= 终端基座域 =================
    namespace 终端基座域 {
        class 物理终端 {
            物理硬件防伪标识(MAC/SN)
            硬件能力规格
            外设连接状态
        }
        class 逻辑身份配置 {
            局域网寻址(IP/计算机名)
            业务归属(教室/座位)
        }
        class 本地资产库 {
            本地基础镜像
            本地快照
        }
        class 运行状态哨兵 {
            网络与硬件负载
        }
    }

    %% ================= 终端引导域 =================
    namespace 终端引导域 {
        class 引导菜单 {
            启动项策略(可见性/默认项)
        }
    }

    %% ================= 终端桌面域 =================
    namespace 终端桌面域 {
        class 运行桌面 {
            当前活动系统
        }
        class 驻留代理端 {
            差异数据同步能力()
            接收管控指令()
        }
        class 教室中控台 {
            局域网群控发令()
        }
    }

    %% ================= 核心业务逻辑连线 (沟通口述辅助线) =================

    %% 交付与建档
    便携部署介质 ..> 物理终端 : [初始化] 裸机刷入底层并灌入镜像
    物理终端 ..> 教室建档清单 : [信息采撷] 脱机完毕提取硬件与位置生成清单
    教室建档清单 ..> 终端云档案 : [联机纳管] 导入平台确立云端唯一身份
    终端云档案 --> 物理终端 : [逻辑映射]

    %% 资产流转
    本地资产库 <..> 云端资产库 : [资产流转] 镜像下发与快照增量双向同步

    %% 身份与运行
    物理终端 *-- 逻辑身份配置 : 承载
    物理终端 *-- 运行状态哨兵 : 承载
    逻辑身份配置 ..> 运行桌面 : [环境注入] OS启动时读取配置获得局域网合法身份

    %% 引导与管控
    引导菜单 --> 本地资产库 : [指向] 确定本次开机加载的目标资产
    运行桌面 *-- 驻留代理端 : [内嵌]
    运行桌面 o-- 教室中控台 : [加装]
    教室中控台 ..> 物理终端 : [底层唤醒] 发送WOL开机魔术包
    教室中控台 ..> 驻留代理端 : [系统级控制] 发送软关机/重启指令
```
