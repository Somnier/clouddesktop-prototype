# 云桌面业务对象关系图 (PM 终极业务推演版)

> **设计思想复盘**：
> 1. **回归最纯粹的业务实体**：彻底删除了“云桌面客户端”、“管控软件”等软件部署层面的服务壳子。将其业务能力还原给真正的业务实体——“学生机桌面”拥有同步状态，“教师机桌面”拥有群控能力。
> 2. **实体拆分的生命周期逻辑**：为什么终端要拆出“物理终端”、“终端业务配置”、“终端运行监控”？因为它们的生命周期和流转方向完全不同：物理硬件是天生的（只读上报），配置是服务器赋予并供桌面OS消费的（注入），监控状态是实时变化用于看板展示的。同理，服务器由于是中心黑盒，其配置与负载合并即可，无需过度设计。
> 3. **找回丢失的核心拼图**：重新引入了“逻辑教室”作为服务器域的核心统筹实体；将“启动菜单”明确为“BIOS引导策略”，并补齐了核心业务属性“数据还原模式”。
> 4. **隔离“辅助实施域”**：明确“安装U盘”和“终端清单”的工具人属性，它们穿插于初始化部署、考试环境搭建等特定环节，而非终端基座的固有组件。

```mermaid
classDiagram
    %% ================= 服务器域 =================
    namespace 服务器域 {
        class 服务器中心 {
            网络地址(IP/域名)
            系统运行负载
        }
        class 授权许可 {
            可用授权点数
        }
        class 逻辑教室 {
            教室名称
            座位矩阵布局
        }
        class 终端云档案 {
            绑定的MAC地址
            绑定的IP与计算机名
            归属的教室与座位
            实时在线状态
        }
        class 告警处置中心 {
            异常类型
            异常源设备(MAC)
            溯源教室与座位
            处理状态
        }
        class 云端基础镜像 {
            OS类型
        }
        class 云端快照 {
            桌面名称
            差异数据包
        }
    }

    %% ================= 辅助实施域 =================
    namespace 辅助实施域 {
        class 安装U盘 {
            预装底层系统
            携带基础镜像
        }
        class 教室终端清单 {
            单教室终端全量数据快照(Excel)
        }
    }

    %% ================= 终端基座域 =================
    namespace 终端基座域 {
        class 物理终端 {
            物理硬件标识(MAC/SN)
            硬件能力规格
            外设连接状态
        }
        class 终端业务配置 {
            分配的局域网IP
            分配的计算机名
            所属教室与座位
        }
        class 终端运行监控 {
            网络联通状态
            CPU与内存负载
        }
        class 本地告警记录 {
            告警类型
            处理状态
        }
        class 本地基础镜像 {
            OS类型
        }
        class 本地快照 {
            快照名称
        }
    }

    %% ================= 终端BIOS域 =================
    namespace 终端BIOS域 {
        class BIOS引导策略 {
            可见启动项列表
            默认启动项
            数据还原模式(每次还原/保留数据)
        }
    }

    %% ================= 终端桌面域 =================
    namespace 终端桌面域 {
        class 学生机桌面 {
            桌面内数据同步状态(上传中/已就绪)
        }
        class 教师机桌面 {
            群控指令下发能力(唤醒/关机)
        }
    }

    %% ================= 核心业务流转接口 (沟通辅助) =================

    %% 宏观统筹与建档
    服务器中心 "1" *-- "n" 逻辑教室
    逻辑教室 "1" o-- "n" 终端云档案 : 建立逻辑包含关系
    安装U盘 ..> 物理终端 : 初始化装机 / 考试环境部署工具
    物理终端 ..> 教室终端清单 : 导出当前硬件与位置信息
    教室终端清单 ..> 终端云档案 : 导入服务器建立档案映射
    终端云档案 --> 物理终端 : 唯一指向

    %% 终端底层的数据解耦与碰撞
    物理终端 *-- 终端业务配置 : 承载
    物理终端 *-- 终端运行监控 : 承载
    物理终端 *-- 本地告警记录 : 离线时硬件变更记账
    本地告警记录 <..> 告警处置中心 : 联机时状态同步与对齐探讨

    %% 资产流转与引导生命周期
    本地基础镜像 ..> 云端基础镜像 : 首次联机备份
    本地快照 <..> 云端快照 : 桌面差异数据双向同步
    BIOS引导策略 --> 本地快照 : 决定开机加载目标与还原规则
    本地快照 --> 学生机桌面 : 实例化为用户运行环境
    本地快照 --> 教师机桌面 : 实例化为教师运行环境

    %% 桌面运行期的依赖
    终端业务配置 ..> 学生机桌面 : 桌面启动时必须读取以获得局域网身份
    终端业务配置 ..> 教师机桌面 : 同上
    教师机桌面 ..> 物理终端 : 局域网发送 WOL 魔术包唤醒硬件
    教师机桌面 ..> 学生机桌面 : 发送系统级关机指令
```
