# 云桌面业务对象关系图

> 本图基于产品经理的纯粹业务视角重构。剥离了技术实现细节（如文件格式、进度条）与非核心层级（如临时测试数据），将聚光灯打在**核心业务实体（终端、教室、镜像、快照、桌面、服务器）** 上，并通过更准确的依赖与从属关系，讲清楚“系统在为谁服务”、“数据从哪里来，要到哪里去”。

---

```mermaid
    namespace 服务器域 {
        class 服务器 {
            IP地址
            域名

            CPU使用率

            有效期限

            告警类型

        }

            账号角色
            管理范围
        }

            OS类型
        }

        }

            网络联机状态
            CPU使用率

            磁盘读写负载

        class 逻辑教室 {
            教室名称

            记录的终端MAC地址
            记录的终端IP地址
            记录的计算机名
            记录的座位号
            记录的硬盘SN码
        }
        class 安装U盘 {
            存储容量
        }
        class 本地基础镜像 {
            OS类型
        }
        class 本地快照 {
            还原模式
        }
        class 终端底层服务 {
            批量部署桌面()
            克隆与还原桌面()
            修改桌面启动项()

        }
        class 操作日志 {
            操作人
            时间
            动作记录
            执行结果
        }
    }

    %% ================= 终端引导启动域 =================

            可用启动项列表

    %% ================= 终端桌面域 =================
    namespace 终端桌面域 {
        class 桌面 {
            桌面名称
        }
        class 云桌面客户端服务 {
            同步本地桌面资产()
            接收关机与重启指令()
        }
        class 教室管控服务 {
            发送一键开机指令()
            发送一键关机与重启指令()
        }
    }

    %% ================= 关系连线 (完整版) =================
    服务器 "1" *-- "1" 服务器运行状态
    服务器 "1" *-- "n" 告警记录
    服务器 "1" o-- "1" 授权许可证书
    服务器 "1" *-- "n" 管理员账号
    服务器 "1" *-- "n" 服务器端终端登记
    服务器 "1" o-- "n" 云端基础镜像
    云端基础镜像 "1" --> "n" 服务器端快照

    服务器 -- 逻辑教室 : 纳管

    逻辑教室 "1" *-- "n" 终端配置
    逻辑教室 "1" *-- "1" 教室终端清单

    终端 "1" *-- "1" 终端配置
    终端 "1" *-- "1" 终端运行状态

    终端底层服务 ..> 终端 : 在裸机上运行
    终端底层服务 "1" *-- "n" 操作日志
    终端底层服务 ..> 教室终端清单 : 产出

    安装U盘 o-- 本地基础镜像
    安装U盘 ..> 终端底层服务 : 为裸机引导安装
    本地基础镜像 "1" --> "n" 本地快照
    终端底层服务 o-- 本地快照 : 操作资产

    教室终端清单 ..> 服务器端终端登记 : 导入建档
    终端 ..> 告警记录 : 触发终端侧异常
    服务器运行状态 ..> 告警记录 : 触发服务器侧异常
    服务器端终端登记 ..> 终端 : 映射物理实体

    本地基础镜像 ..> 云端基础镜像 : 同步
    本地快照 ..> 服务器端快照 : 双向同步

    终端底层服务 ..> 启动菜单 : 业务配置
    启动菜单 --> 本地快照 : 指向引导
    本地快照 --> 桌面 : 加载运行

    桌面 "1" *-- "1" 云桌面客户端服务
    桌面 "1" o-- "1" 教室管控服务

    云桌面客户端服务 ..> 服务器端快照 : 执行资产上传
    云桌面客户端服务 <.. 服务器 : 下发指令
    教室管控服务 ..> 云桌面客户端服务 : 局域网下达控制
    教室管控服务 ..> 终端 : 发送WOL物理唤醒
```

## 版本二：沟通效率 MAX 极简版 (纯粹聚焦拆分与归属)
