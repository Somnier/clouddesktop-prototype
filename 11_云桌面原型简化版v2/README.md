# 云桌面管理系统原型 v2

> 云桌面终端管理系统的交互原型——终端侧 + 平台侧双端完整交互，四端同屏实时同步。
> 技术栈：Node.js + Express + SSE + 原生 HTML/JS，可选 Electron 桌面窗口。

## 快速启动

```bash
npm install
npm run serve          # 启动 Web 服务器
# 浏览器打开 http://localhost:3920/director
```

或 `npm start` 同时启动 Web + Electron 窗口。端口默认 `3920`，可通过环境变量 `PORT` 修改。

## 四端同屏

原型同时运行 4 个端，共享一份 JSON 状态，通过 SSE（600ms tick）实时同步：

| 端 | 地址 | 说明 |
|---|---|---|
| 导演台 | http://localhost:3920/director | 原型操控中心：切换教室 / 触发动作 / 模拟故障 / 重置数据 |
| 母机终端 | http://localhost:3920/terminal/mother | 终端管理软件完整功能（深色工业风） |
| 受控终端 | http://localhost:3920/terminal/controlled | 受控端任务屏（部署 / 同传进度） |
| 管理平台 | http://localhost:3920/platform | Web 管理后台（三栏布局 + 数据图表） |

启动后打开**导演台**，切换不同教室即可在各端同步观察对应状态。导演台的测试按钮可快捷跳转到各功能页面。

## 终端侧功能

母机终端是操作员的主界面，覆盖教室管理全流程：

| 功能区 | 说明 |
|---|---|
| 首页 | 6 个功能入口（设置本机 / 管理桌面 / 网络同传 / 替换故障终端 / 重置终端 / 考试桌面） |
| 设置本机 | 本机信息 + 网络配置，服务器地址失焦自动检测连通性 |
| 管理桌面 | 卡片网格 + "+"导入卡，溢出菜单（编辑/复制/设为默认/隐藏/删除），磁盘占比在标题栏 |
| 统一工作台 | 双 Tab 设计——**设置布局**（发现+网格+IP/DNS规则+完成布局）/ **教室维护**（部署 / IP / 导出） |
| 座位网格 | 替代终端列表，布局页显示发现终端MAC，方块 2 态切换（可用 ↔ 禁用） |
| 故障处理 | 教室网格选座 + 一键替换 / 重置，进度实时反馈 |
| 终端发现 | `tickRecover` 逐台上线模拟（每 tick 1-2 台），接管后自动扫描 |

### 工作台交互流程

```
首页 → 网络同传 → 工作台「设置布局」Tab
  ├─ 扫描网络 → 配置IP/DNS/机器名规则 → 预览网格(终端MAC) → 完成布局
  ├─ 完成布局后自动分配所有终端 → 跳转「教室维护」Tab
  └─ 可随时切回「设置布局」重新调整 → 「教室维护」→ 部署 / IP修改 / 导出清单
```

## 平台侧功能

Web 管理后台采用三栏布局（侧边栏 + 主内容 + 右侧信息面板）：

| 功能区 | 说明 |
|---|---|
| 仪表盘 | 校区总览指标 + 健康度饼图（健康 / 警告 / 严重三色分布）+ 服务器监控折线图 + 告警日志 |
| 教室列表 | 已部署教室一览，导入终端清单（按名称自动匹配教室） |
| 教室详情 | 双栏：总览（指标 + 完整告警中心含排序）+ 终端列表（4 功能区 + 按用途快速选择） |
| 终端操作 | 教室控制（关机/重启/USB/外网）、教室部署（发桌面/改IP）、教室测试、跨教室广播隔离测试 |
| 桌面资产 | 卡片式展示（定位教室 + 终端引用可展开）+ 排序 + 空闲桌面可删除 + 导入/导出 |
| 服务器地址 | 一级入口变更管理服务器地址 |

## 数据与种子

| 文件 | 说明 |
|---|---|
| `data/seed.json` | 种子数据：2 个校区、6 间教室（1 间空白 + 5 间已部署）、约 280 台终端、桌面资产、告警 |
| `data/state.json` | 运行时状态快照，删除后重启自动从 seed 重建 |

焦点教室 `yp-a301`（A301 公共教学机房）初始状态为 `blank`——无终端、无桌面、无网络配置，用于演示从零接管到完整部署的全流程。其余 5 间教室为 `deployed` 状态，终端全部在线。

导演台「重置到初始种子数据」按钮可随时回到初始状态。

## 技术架构

```
浏览器 (4端)          server.mjs
┌──────────┐     ┌──────────────────┐
│ SSE 订阅  │◄────│ 600ms tick 循环   │  状态变更 → 标记 dirty → 每 5 tick 广播
│ POST 动作 │────►│ action handler   │  收到动作 → 修改 state → 立即广播
│ innerHTML │     │ state.json 持久化 │
└──────────────┘     └──────────────────┘
```

- **SSE 实时同步**：服务端 600ms 一次 tick，状态变更后广播全量 JSON 给所有连接的客户端
- **渲染去重**：`_stateFingerprint()` 仅提取结构性字段（排除 metrics/heartbeat 波动），指纹不变时跳过渲染；`requestAnimationFrame` 合并同帧内的多次 SSE 消息
- **输入保护**：渲染前缓存所有 `<input>` / `<textarea>` 的值和光标位置，`innerHTML` 替换后恢复，`_isRendering` 标志位防止 blur 级联
- **滚动保护**：渲染前记录所有可滚动容器的 `scrollTop`，渲染后恢复
- **tick 模拟**：`tickRecover` 逐台上线、`tickDeploy` 部署进度、`tickTransmit` 同传进度，驱动终端状态变化

## v2 与 v1 的关键差异

| 维度 | v1（10 版） | v2（11 版） |
|---|---|---|
| 教室接管 | 独立 `takeoverScreen` → 确认 → 跳转工作台 | 直接进入工作台，「设置布局」单页完成发现+规则+完成布局 |
| 工作台 | 功能按钮列表，点击跳子页面 | 双 Tab（设置布局 / 教室维护），就地操作 |
| 维护操作 | 跳转到独立页面（IP / 部署 / 桌面） | Tab 内切换，设置状态保留 |
| SSE 重绘 | 全量 `innerHTML`，输入值丢失 | 状态指纹去重：仅结构性字段变更才重绘，排除 metrics/heartbeat 波动；`_inputCache` + `_isRendering` 保护输入 |
| 种子教室 | `deployed`，终端全在线 | `blank`，终端逐台上线 |
| 平台 | 基础列表 | 三栏布局 + 健康饼图 + 4 功能分区 + 卡片式桌面资产 |
| 退出按钮 | `btn-danger` 红色填充 | `btn-exit` 红色描边 + hover 实心填充白字 |
