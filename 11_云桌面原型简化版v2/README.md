# 云桌面管理系统原型 v2

> 云桌面终端管理系统的交互原型，用于验证终端侧操作流程与页面设计。
> 技术栈：Node.js + Express + SSE 实时同步 + 原生 HTML/JS 前端，可选 Electron 桌面窗口。

## v2 相对于 v1 的主要改动

- **首页按钮统一**：全部使用动名词命名（设置本机、管理桌面、网络同传…），样式统一，取消分组差异
- **退出按钮醒目**：顶栏右上角红色退出按钮，一眼可见
- **服务器面板实时感知**：输入服务器地址失焦即自动检测连接，显示转圈动画
- **桌面卡片重设计**：明确"桌面占用 XX GB（系统盘 + 数据盘）"，状态与操作按钮分区，未同步桌面红色警示
- **存储数据真实化**：系统占用 ~35 GB，每个桌面有真实磁盘用量值（diskSize），各桌面之和 = 存储统计
- **导入流程统一**：镜像和桌面包导入均经创建流程，部署方式优先于数据盘，无数据盘时隐藏大小选项
- **导入导出 Electron 化**：支持 Electron 系统级文件选择对话框，默认定位 U 盘路径
- **故障替换/重置**：教室布局网格选座（仅显示座位号），替换和重置过程显示进度条 + 警告文字
- **输入焦点修复**：render 重绘时保留正在编辑的输入值，防止输入 1 字符后丢失焦点
- **取消/确认全局统一**：所有弹框和表单均遵循"取消在左、确认在右"
- **按钮尺寸统一**：移除小按钮（btn-sm），保持所有操作按钮可点击性
- **工作台优化**：设置布局切入时自动重新扫描终端，未接管不可切换到教室维护

## 设计目标

- **终端侧为主**：聚焦母机操作员视角，覆盖教室接管、桌面管理、批量部署、IP 维护、故障处理全流程
- **网格化布局**：用座位网格替代终端列表，所有操作围绕真实教室布局展开
- **统一工作台**：接管 + 布局 + 维护操作整合为两页（设置布局 / 教室维护），减少页面跳转
- **可演示**：导演台一键切换教室状态，四端同屏实时同步

## 主要功能

| 功能 | 说明 |
|---|---|
| 教室接管 | 扫描网络 → 确认接管 → 自动生成座位网格 |
| 桌面管理 | 查看/创建/复制/删除/上传桌面，支持数据盘，文件夹选择器模拟 |
| 教室部署 | 4 步向导：母机准备 → 座位网格标记 → 终端绑定 → 部署传输 |
| 统一工作台 | 设置布局页（接管+网格+IP规则）+ 教室维护页（部署/IP修改/桌面管理/导出） |
| IP 维护 | 批量修改终端 IP / 服务器地址，拖拽排序 |
| 故障处理 | 一键替换（教室网格选座）+ 一键重置 |
| 导出清单 | 导出终端清单为文件 |

## 四端同屏

原型同时运行 4 个端，共享 JSON 状态，通过 SSE 实时同步：

| 端 | 地址 | 说明 |
|---|---|---|
| 导演台 | http://localhost:3920/director | 原型控制台：切换教室、触发动作、模拟故障、重置数据 |
| 母机终端 | http://localhost:3920/terminal/mother | 终端管理系统完整功能 |
| 受控终端 | http://localhost:3920/terminal/controlled | 受控终端任务屏 |
| 管理平台 | http://localhost:3920/platform | Web 管理平台 |

## 快速启动

```bash
npm install

# 方式一：仅 Web 服务器
npm run serve
# 浏览器打开 http://localhost:3920/director

# 方式二：Web + Electron 窗口
npm start
```

端口默认 `3920`，可通过环境变量 `PORT` 修改。

## 数据

- `data/seed.json` — 种子数据（6 间教室、282 台终端、告警、桌面）
- `data/state.json` — 运行时快照，删除后重启自动重建
- 导演台「重置到初始种子数据」可随时恢复

## 目录结构

```
├── server.mjs              # Express 服务器 + 状态管理 + SSE
├── start.mjs               # 启动脚本
├── data/
│   └── seed.json           # 种子数据
├── public/
│   ├── shared/             # 共享：base.css, ui.js, model.js, state-client.js
│   └── apps/
│       ├── director/       # 导演台
│       ├── terminal/       # 终端（母机 + 受控）
│       └── platform/       # 管理平台
└── electron/               # Electron 入口
```

## 使用

1. 启动后打开**导演台**（原型操控中心）
2. 切换不同教室查看各阶段状态，终端自动同步
3. 导演台测试流程按钮可快速跳转到各功能页面
4. 终端座位网格可点击切换观察对象
5. 所有操作实时同步到四个端
