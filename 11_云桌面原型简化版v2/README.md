# 云桌面管理系统原型 v2

> 云桌面终端管理系统的交互原型，用于验证终端侧操作流程与页面设计。
> 技术栈：Node.js + Express + SSE 实时同步 + 原生 HTML/JS 前端，可选 Electron 桌面窗口。

## v2 相对于 v1 的主要改动

### 一、架构级差异

| 维度 | v1 (10 版) | v2 (11 版) |
|---|---|---|
| 教室接管 | 独立 `takeoverScreen` 页面，确认后跳转工作台 | 接管整合进工作台「设置布局」Tab，无独立页面 |
| 工作台 | 信息卡片 + 功能按钮列表，点击跳转到子页面 | 双 Tab 页（设置布局 / 教室维护），所有操作就地完成 |
| 部署流程 | 4 步向导（母机准备 → 网格 → 绑定 → 传输），每步独立页面 | 4 步向导仍保留但主入口改为工作台维护 Tab 内嵌操作 |
| IP 维护 | 独立 `maintIpScreen` 页面 | 可内嵌到工作台维护 Tab（从工作台触发则留在工作台） |
| SSE 重绘 | 全量 `innerHTML` 替换，编辑中输入值丢失 | 增加 `_inputCache` + `_restoreAllInputs()` + 滚动位置保存/恢复 |
| 种子数据 | 焦点教室 `stage:'deployed'`，终端全部在线 | 焦点教室 `stage:'blank'`，终端初始离线，母机固定在线 |
| 终端发现 | 终端静态存在 | `tickRecover` 逐台上线模拟（每 tick 1-2 台） |

### 二、终端侧画面对照

| 画面 | v1 | v2 | 变化说明 |
|---|---|---|---|
| `homeScreen` | 6 个功能大按钮 | 6 个动名词按钮 | 文案统一（设置本机、管理桌面、网络同传…） |
| `takeoverScreen` | 独立扫描页 + MAC 列表 + 确认接管 | 重定向到 `workbenchScreen` | 接管逻辑融入布局 Tab |
| `workbenchScreen` | 教室信息卡片 + 告警面板 + 按钮行 + 终端列表 | 双 Tab（设置布局 / 教室维护）+ 左设置右网格 | 全面重设计 |
| `wbLayoutContent` | 不存在 | 左 5 卡片 + 右 3 区（指南/网格/操作） | v2 新增 |
| `wbMaintContent` | 不存在 | 左功能卡片 + 右网格进度 | v2 新增 |
| `localDesktopScreen` | 基础桌面列表 | 重设计：存储环、详细桌面卡片、导入流程 | 增加存储可视化 |
| `localNetworkScreen` | 基础输入表单 | 增加连接检测动画（失焦即检测） | 交互增强 |
| `faultReplaceScreen` | 终端列表选择 | 教室网格选座 + 进度条 | 交互方式变化 |

### 三、交互模型差异

#### 接管 + 布局流程

- **v1**：首页 → `takeoverScreen`（扫描 + 确认） → 工作台 → 点击「教室维护」→ deploy 4 步向导
- **v2**：首页「网络同传」→ 工作台布局 Tab（扫描 + 设规则 + 确认接管 → 绑定阶段 → 切换到维护 Tab 部署）

#### 网格操作

| 操作 | v1 | v2 |
|---|---|---|
| 方块点击 | 3 态循环：可用 → 禁用 → 删除 → 可用 | 2 态切换：可用 ↔ 禁用 |
| 绑定切换 | 部署模式下点击已绑定方块可解绑 | 同上，解绑后点击可重新绑定 |
| 禁用跳过 | 禁用方块后终端自动顺延 | 同 v1 |

#### 维护模式

| 行为 | v1 | v2 |
|---|---|---|
| 模式切换 | 点击按钮跳转到独立页面 | Tab 内切换，右侧网格就地更新 |
| 设置保留 | 切换到其他页面丢失 | 切换功能后返回保留（open-maint-ip 判断 draft 是否已有值） |
| 任务进度 | 跳转到 taskProgressScreen 独立页 | 内嵌在工作台网格区，功能切换已锁定 |

### 四、UI / 样式差异

| 改动 | 说明 |
|---|---|
| 退出按钮 | v1 用 `btn btn-danger btn-sm` 红色填充按钮；v2 用 `btn-exit` 透明底 + 边框 + hover 变红 |
| 弹框确认按钮 | v1 全部 `btn-danger`（红色）；v2 区分破坏性/非破坏性，导入 / 导出 / 退出用 `btn-primary`（蓝色） |
| 状态标签 | v1 用标准 `pill()` 与按钮等权；v2 增加 `pill-sm` 小号标签（.68rem, opacity .75），降低视觉权重 |
| 存储圆环 | v1 桌面占用 0 GB 时仍显示蓝色弧（stroke-linecap 导致）；v2 为 0 时不渲染弧 + 图例色块变边框色 |
| 按钮尺寸 | v2 移除 `btn-sm`，所有按钮保持统一可点击高度 |
| 连接检测 | v2 在服务器地址输入框 blur 时自动触发检测动画，不需要单独点按钮 |

## 设计目标

- **终端侧为主**：聚焦母机操作员视角，覆盖教室接管、桌面管理、批量部署、IP 维护、故障处理全流程
- **网格化布局**：用座位网格替代终端列表，所有操作围绕真实教室布局展开
- **统一工作台**：接管 + 布局 + 维护操作整合为两页（设置布局 / 教室维护），减少页面跳转
- **可演示**：导演台一键切换教室状态，四端同屏实时同步

## 主要功能

| 功能 | 说明 |
|---|---|
| 教室接管 | 扫描网络 → 确认接管 → 自动生成座位网格 |
| 桌面管理 | 查看/创建/复制/删除/上传桌面，支持数据盘，文件夹选择器模拟 |
| 教室部署 | 4 步向导：母机准备 → 座位网格标记 → 终端绑定 → 部署传输 |
| 统一工作台 | 设置布局页（接管+网格+IP规则）+ 教室维护页（部署/IP修改/桌面管理/导出） |
| IP 维护 | 批量修改终端 IP / 服务器地址，拖拽排序 |
| 故障处理 | 一键替换（教室网格选座）+ 一键重置 |
| 导出清单 | 导出终端清单为文件 |

## 四端同屏

原型同时运行 4 个端，共享 JSON 状态，通过 SSE 实时同步：

| 端 | 地址 | 说明 |
|---|---|---|
| 导演台 | http://localhost:3920/director | 原型控制台：切换教室、触发动作、模拟故障、重置数据 |
| 母机终端 | http://localhost:3920/terminal/mother | 终端管理系统完整功能 |
| 受控终端 | http://localhost:3920/terminal/controlled | 受控终端任务屏 |
| 管理平台 | http://localhost:3920/platform | Web 管理平台 |

## 快速启动

```bash
npm install

# 方式一：仅 Web 服务器
npm run serve
# 浏览器打开 http://localhost:3920/director

# 方式二：Web + Electron 窗口
npm start
```

端口默认 `3920`，可通过环境变量 `PORT` 修改。

## 数据

- `data/seed.json` — 种子数据（6 间教室、282 台终端、告警、桌面）
- `data/state.json` — 运行时快照，删除后重启自动重建
- 导演台「重置到初始种子数据」可随时恢复

## 目录结构

```
├── server.mjs              # Express 服务器 + 状态管理 + SSE
├── start.mjs               # 启动脚本
├── data/
│   └── seed.json           # 种子数据
├── public/
│   ├── shared/             # 共享：base.css, ui.js, model.js, state-client.js
│   └── apps/
│       ├── director/       # 导演台
│       ├── terminal/       # 终端（母机 + 受控）
│       └── platform/       # 管理平台
└── electron/               # Electron 入口
```

## 使用

1. 启动后打开**导演台**（原型操控中心）
2. 切换不同教室查看各阶段状态，终端自动同步
3. 导演台测试流程按钮可快速跳转到各功能页面
4. 终端座位网格可点击切换观察对象
5. 所有操作实时同步到四个端
