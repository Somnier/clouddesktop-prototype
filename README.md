# 云桌面管理系统 — 原型设计项目

[![License: MIT](https://img.shields.io/badge/License-MIT-blue.svg)](LICENSE)

一个教学云桌面管理系统的产品设计与交互原型项目，用于研究和学习如何通过深度人机协同（AI Agent 辅助）完成从原始需求到可运行原型的全流程。

项目以某教学场景为背景，覆盖终端侧（母机管理软件）和平台侧（Web 管理平台）两大端，模拟管理多校区、多教室、多终端的云桌面生命周期。

---

## 项目做了什么

本项目完整记录了一个中等复杂度产品从零到一的原型设计过程，包含从客户前期沟通到可运行原型交付的全部阶段。

项目从《原型设计工程实践指南》提出的五阶段方法展开——原始资料整理、事实层固化、页面结构设计、可交互原型实现、评审与修正——实际推进中，每一阶段的产出都沉淀在对应的目录里，形成了完整的工程演进链条。

### 各阶段产出

1. **前期沟通与原始资料**（01 目录）：项目起点是客户侧的技术方案讨论和会议记录，包括系统架构说明、终端能力边界和初始功能需求。这些原始资料是后续所有推导的事实来源，保持原样不修改。
2. **业务建模与需求收敛**（03→04 目录）：从原始资料出发，经过 11 轮迭代（v1→v11），将模糊的需求逐步收敛为结构化的业务对象关系图和 PRD，冻结为正式基线。
3. **事实沉淀**（05 目录）：在页面设计过程中，把跨页面、跨端反复出现的规则提取为独立事实文档。事实层先于页面层固化，确保后续页面不会各自为政。
4. **页面设计**（06 + 07 目录）：终端侧 7 个页面（TERM-01 ~ TERM-07）、平台侧 6 个页面（WEB-01 ~ WEB-05 + WEB-02A），每个页面包含信息架构、交互流程和 ASCII 线框图。事实层和页面层交替推进，每轮跨层校验后回修。
5. **可运行原型**（09 + 10 目录）：用 Node.js + Express + SSE 构建四端同屏交互原型，评审后持续修正。
6. **方法论输出**（02 目录）：沉淀了两份通用工程方法指南（不依赖具体项目），以及一份对当前 AI Agent 工作模式的分析记录。

## 产品设计概要

### 系统定位

不是重型 VDI，而是以终端本地运行能力为基础、以服务器统一纳管能力为增强的云桌面教学方案。核心设计原则：

- 支持脱机交付：网络未就绪时，单教室仍可独立完成部署
- 联机后增强：终端状态、桌面资产和异常信息逐步纳入统一管理
- 多教室复用：重复单教室流程为基础路径，服务器辅助和离线打包并列提效
- 交付后自助：机房管理员在远程指导下即可完成常见运维

### 终端侧页面（06 目录）

终端管理软件运行在教室母机上，共 7 个页面：

| 编号 | 页面 | 说明 |
|---|---|---|
| TERM-01 | 管理系统首页 | 教室状态总览、终端列表、网络与服务器配置 |
| TERM-02 | 母机接管页 | 接管教室、释放教室、换母机识别 |
| TERM-03 | 母机工作页 | 桌面制作、批量日常操作的中枢 |
| TERM-04 | 单教室部署 | 3 步向导：选来源 → 选桌面 → 选目标终端 → 执行 |
| TERM-05 | 教室维护 | 加装软件、IP 修改、教室终端清单导出 |
| TERM-06 | 故障处理 | 一键替换、一键重置、手动修正 |
| TERM-07 | 考试桌面 | 考试启动设置与考后恢复 |

### 平台侧页面（07 目录）

Web 管理平台运行在服务器上，共 5 个主页面 + 1 个详情子页：

| 编号 | 页面 | 说明 |
|---|---|---|
| WEB-01 | 平台总览 | 校区级指标、服务器状态、任务进度、异常与风险 |
| WEB-02 | 教室管理 | 教室列表、终端清单导入、批量工具栏（分 4 组：单教室操作、单教室测试、多教室操作、多教室测试） |
| WEB-02A | 终端详情 | 单终端配置、硬件、网络、桌面、告警全量信息 |
| WEB-03 | 桌面资产 | 校区级桌面管理，支持桌面包导入导出和无引用桌面清理 |
| WEB-04 | 告警中心 | 按教室/类型筛选与排序、跳转教室和终端 |
| WEB-05 | 系统运维 | 授权管理、服务器地址变更、校区与账号管理 |

### 事实层（05 目录）

设计过程中沉淀的业务事实，供所有页面引用而不是各自重复推导：

- 母机接管与教室绑定机制
- 批量部署范围与终端状态同步机制
- 任务执行中断与受控终端任务屏机制
- 终端与服务器按需同步
- 平台侧页面范围与任务边界
- 跨端页面表达与字段口径

## 交互原型

原型采用四端同屏架构，共享同一份 JSON 状态，通过 SSE 实时同步：

| 端 | 地址 | 说明 |
|---|---|---|
| 导演台 | `http://localhost:3920/director` | 测试控制台：切换场景、触发动作、模拟故障 |
| 母机终端 | `http://localhost:3920/terminal/mother` | 终端管理软件的完整交互 |
| 受控终端 | `http://localhost:3920/terminal/controlled` | 受控终端任务屏 |
| 管理平台 | `http://localhost:3920/platform` | Web 管理平台 |

### 快速启动

```bash
cd 09_云桌面原型
npm install
npm run serve          # Web 服务器（浏览器访问）
# 或
npm start              # Web + Electron 桌面窗口
```

种子数据（`data/seed.json`）包含模拟的教室和终端数据，导演台可随时重置。

### 方法论产出（02 目录）

本项目在推进过程中沉淀了三份方法类文档：

- **原型设计工程实践指南**：从原始资料到可运行原型的五阶段工程方法（原始资料整理 → 事实层固化 → 页面结构设计 → 可交互原型实现 → 评审与修正）。本项目的目录结构和推进节奏即按此方法组织。
- **深度人机协同工程实践指南**：与 AI Agent 进行多轮、多文件、多阶段深度协作的通用工程方法论（事实分层、锚点机制、偏移控制、边界定义）。
- **AI Agent 工作模式说明**：对当前 VS Code + Copilot Agent 环境的工程化分析，记录了从纯对话到 Agent 编排的能力落地路径、工具体系、记忆机制和实际工作流约束。

前两份指南不依赖具体项目，可直接复用于其他原型设计和人机协同场景。
