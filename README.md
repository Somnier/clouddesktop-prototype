# 云桌面管理系统 — 产品原型设计

[![License: MIT](https://img.shields.io/badge/License-MIT-blue.svg)](LICENSE)

本项目是**云桌面管理系统的完整产品原型设计**——从原始需求到可运行的交互原型，覆盖终端侧（母机管理软件）和平台侧（Web 管理平台），管理多校区、多教室、多终端的桌面生命周期。原型以导演台、母机终端、受控终端、管理平台四端同屏运行，共享同一份状态并通过 SSE 实时同步。

云桌面产品的设计方向是通用终端管理，初始需求仅研究了高校教学场景。

---

## 项目结构

项目按"原始资料 → 正式基线 → 事实层（业务事实 + 设计事实）→ 页面设计层 → 原型 → 研发落地"的顺序递进，每一层的产出沉淀在对应目录：

```
├── 01_原始资料/               # 客户侧原始需求清单
├── 02_正式基线/               # PRD、业务对象关系图、初始需求清单（冻结基线）
├── 03_云桌面业务事实/         # 核心概念与统一口径 + 跨页面/跨端业务机制
├── 04_云桌面设计事实/         # UI 呈现原则、交互事实、批量部署交互、导演台
├── 05_终端侧页面设计/         # 终端管理软件 TERM-01 ~ TERM-07
├── 06_平台侧页面设计/         # Web 管理平台 WEB-01 ~ WEB-05 + WEB-02A
├── 07_可行性与工程筹备/       # 方法指南、可行性分析、原型范围取舍、交接精要
│   ├── 方法指南/              #   原型设计与人机协同的通用方法论
│   └── 可行性分析/            #   技术背景、研发可行性、纯前端原型可行性
└── 08_云桌面原型/             # 可运行交互原型（Node.js + Express + SSE + Electron）
```

设计阶段的递进关系：01 原始资料经业务建模收敛为 02 正式基线；03 业务事实与 04 设计事实并列，分别沉淀"系统机制 / 口径"与"界面呈现 / 交互方式"；05 终端侧、06 平台侧页面设计是前期设计完成的阶段。**03 业务事实 + 04 设计事实 + 05/06 页面设计共同直接指导了 08 云桌面原型的代码实现。** 设计文档保留完整设计，原型按 [云桌面原型范围取舍](07_可行性与工程筹备/云桌面原型范围取舍.md) 的"做/不做"取舍实现。原型落地前的可行性论证与方法论存档于 07_可行性与工程筹备/。

## 产品设计概要

### 系统定位

不是重型 VDI，而是以终端本地运行能力为基础、以服务器统一纳管能力为增强的云桌面方案。核心设计原则：

- 支持脱机交付：网络未就绪时，单教室仍可独立完成部署
- 联机后增强：终端状态、桌面资产和异常信息逐步纳入统一管理
- 多教室复用：重复单教室流程为基础路径，服务器辅助和离线打包并列提效
- 交付后自助：机房管理员在远程指导下即可完成常见运维

### 终端侧页面（05 目录）

终端管理软件运行在教室母机上，共 7 个页面：

| 编号 | 页面 | 说明 |
|---|---|---|
| TERM-01 | 管理系统首页 | 教室状态总览、终端列表、网络与服务器配置 |
| TERM-02 | 母机接管页 | 接管教室、释放教室、换母机识别 |
| TERM-03 | 母机工作页 | 桌面制作、批量日常操作的中枢 |
| TERM-04 | 单教室部署 | 3 步向导：选来源 → 选桌面 → 选目标终端 → 执行 |
| TERM-05 | 教室维护 | 加装软件、IP 修改、教室终端清单导出 |
| TERM-06 | 故障处理 | 一键替换、一键重置、手动修正 |
| TERM-07 | 考试桌面 | 考试启动设置与考后恢复（设计保留，原型未实现，详见 04 迭代取舍） |

### 平台侧页面（06 目录）

Web 管理平台运行在服务器上，共 5 个主页面 + 1 个详情子页：

| 编号 | 页面 | 说明 |
|---|---|---|
| WEB-01 | 平台总览 | 校区级指标、服务器状态、任务进度、异常与风险 |
| WEB-02 | 教室管理 | 教室列表、终端清单导入、批量工具栏（分 4 组：单教室操作、单教室测试、多教室操作、多教室测试） |
| WEB-02A | 终端详情 | 单终端配置、硬件、网络、桌面、告警全量信息 |
| WEB-03 | 桌面资产 | 校区级桌面管理，支持桌面包导入导出和无引用桌面清理 |
| WEB-04 | 告警中心 | 按教室/类型筛选与排序、跳转教室和终端 |
| WEB-05 | 系统运维 | 授权管理、服务器地址变更、校区与账号管理 |

### 事实层（03 业务事实 + 04 设计事实）

设计过程中沉淀的事实，供所有页面引用而不是各自重复推导：

**业务事实（03）**——系统机制与口径：

- 核心概念与统一口径（总纲）
- 母机接管与教室绑定机制
- 单教室批量部署机制
- 任务执行中断与受控终端任务屏机制
- 数据上传与一致性机制（跨端）
- 终端导出与平台导入机制（跨端）
- 字段口径与术语说明（跨端）
- 镜像-快照-桌面三层资产模型（跨端）

**设计事实（04）**——界面呈现与交互方式：

- UI 呈现原则（设计令牌、排版基准、信息顺序、表单/弹框/按钮规范、导入导出、实时预览）
- 交互事实（设置本机、桌面管理、网络同传三步主流程、演示与产品分离等）
- 批量部署交互设计（座位网格、多步导航、过程与结果就地切换、逐台确认呈现）
- 导演台设计事实（导演台的演示定位、职责与边界）

## 交互原型

导演台、母机终端、受控终端、管理平台四端共享同一份 JSON 状态，通过 SSE 实时同步：

| 端 | 地址 | 说明 |
|---|---|---|
| 导演台 | `http://localhost:3920/director` | 测试控制台：切换场景、触发动作、模拟故障 |
| 母机终端 | `http://localhost:3920/terminal/mother` | 终端管理软件的完整交互 |
| 受控终端 | `http://localhost:3920/terminal/controlled` | 受控终端任务屏 |
| 管理平台 | `http://localhost:3920/platform` | Web 管理平台 |

### 快速启动

```bash
cd 08_云桌面原型
npm install
npm run serve          # Web 服务器（浏览器访问）
# 或
npm start              # Web + Electron 桌面窗口
```

> 09、10 目录是更早版本的原型，11 是当前迭代版本。

种子数据（`data/seed.json`）包含模拟的教室和终端数据，导演台可随时重置。
