# 云桌面跨端结论_页面表达与字段口径说明

> 文档角色：03_云桌面业务事实 的跨端结论之一。
> 作用：约定终端侧与平台侧共用的数据分层、字段口径与术语边界，使两端面对同一批对象时表达一致。
> 前置：术语与双端/双模式概念见 [云桌面核心概念与统一口径.md](./云桌面核心概念与统一口径.md)。

---

## 1. 数据分层

页面数据至少分为五层，任一字段只归属一层，不可用一条字段混表多层含义：

1. **当前实时状态**：当前在线、当前机器名、当前 IP、当前工作 MAC、当前连通性。
2. **终端业务配置**：教室、座位号、终端用途、机器名、网络配置（IP/子网掩码/网关/DNS）、服务器地址、默认启动项、还原方式。
3. **当前任务数据**：本轮勾选哪些终端、处理顺序、准备写入什么。
4. **当前任务运行状态**：当前步骤、进度、失败原因、同步状态。
5. **平台侧/服务器侧汇总**：终端登记信息、桌面资产状态、平台汇总。

口径约定：

- **配置**指应长期保存的持久业务值或登记信息；**设置**指用户当前正在选择、确认或准备写入的任务参数。二者不可混用——"设置"未落库前不等于已生效的"配置"。
- "平台侧"与"服务器侧"指同一端（见核心概念文档）；"上传到平台"即"上传到服务器"。

## 2. 三态字段不可混写

凡同时涉及"现在是什么 / 长期应是什么 / 这次准备改成什么"的字段，必须分三层表达：当前生效值、业务配置值、本次任务写入值。IP、机器名、默认启动项、还原方式、终端用途均属此类。页面可并排对照，不可合并为一个模糊字段。

这组字段在批量场景与单机场景表达的是同一组业务字段——区别只在"整批写入"还是"单台处理"，不是字段本身不同。

## 3. 母机视角与受控视角的边界

母机视角关注整间教室、整批终端、整轮任务结果；受控视角只关注本机是否被纳入、执行到哪一步、是否保持现状。同一轮任务下，两者是同一批对象的不同切面，须各自表达、口径一致。

受控期间，受控终端不得自行跳转到单机故障处理（替换/重置/改本机设置），否则控制关系被打断；单机故障处理只在终端未受控、任务已中断/取消、或该终端已明确脱离任务后才成立。

## 4. 可见性与标识口径

- **MAC 地址**是系统内部关键标识，必须保留；但用户阅读困难，界面主路径以座位号、机器名替代，详情/诊断场景再展示完整 MAC。
- **座位号 / 机器名 / IP** 三者并列展示时顺序恒为：座位 → 机器名 → IP（细则见 04 设计事实）。
- **座位号**为字母+数字连写、不含横杠（如 A01、T01）；**机器名**为教室前缀与座位号以横杠连接（如 D301-A01）；二者为独立字段，座位号不继承机器名格式。
- **桌面同步状态**统一为"未同步 / 同步中 / 已同步"三态；"未同步"需显著提示，避免被误认为已可在服务器侧复用。

## 5. 桌面集合可多选，单桌面动作仍是单选

- 桌面下发集合（批量部署 / 传输）可多选；默认启动桌面从已选集合中再选 1 个。
- 复制桌面、进入桌面编辑是单桌面动作，仅在当前单选 1 个桌面时可用；新建桌面不依赖来源，但完成后处理对象仍是一个具体桌面。
- 不另造"当前操作桌面"这类中间概念制造歧义。

## 6. 三类终端集合不可混为一谈

单教室任务里须能同时表达：

1. **教室受管集合**：母机当前管理的整批终端（含教师终端、待单独处理终端）。
2. **本轮勾选终端**：本次任务真正处理的子集。
3. **本轮保持现状终端**：仍属本教室、本次不参与的终端——显示为"本次未纳入"，并回显其沿用的座位 / 机器名 / IP / 用途等当前内容，不显示为掉线或失败。

## 7. 批量任务须有逐台确认层

批量改写终端配置、桌面或启动设置前，不在"设置完规则"后直接执行，须先把规则展开为逐台写入结果供核对 / 微调，确认后才分发。部署、维护、考试设置均遵循此层。

## 8. 本地完成与服务器同步是两条并行线

终端侧本地任务是否完成先看本地结果；服务器是否已接收状态 / 配置 / 桌面资产是另一条同步线。本地交付可先按本地结果成立；服务器上传影响平台侧可见性与跨教室复用，但不反向推翻终端本地已完成的结果。

## 9. 人工勾选列表支持排序与分页

凡存在人工勾选终端 / 桌面 / 教室的列表，都应支持排序与分页；可排序字段、默认排序与分页策略按页面对象类型与数据量分别确定，不在跨页规则层写死。
