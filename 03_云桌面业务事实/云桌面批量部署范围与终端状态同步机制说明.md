# 云桌面批量部署范围与终端状态同步机制说明

> 文档角色：业务事实层的终端结论之一。
> 作用：约定单教室批量任务中"受管集合 / 本轮勾选 / 保持现状"三类终端的关系，以及母机视角与受控视角的同步口径。
> 前置：双端 / 双模式与"网络同传"口径见 [云桌面核心概念与统一口径.md](./云桌面核心概念与统一口径.md)；字段分层见 [云桌面页面表达与字段口径说明.md](./云桌面页面表达与字段口径说明.md)。

---

## 1. 为什么要统一这组口径

一轮单教室批量任务（即网络同传主流程）开始后，母机视角与受控终端视角看到的是同一批对象的不同切面。若两侧口径不一致，容易出现“同一终端在母机端算已完成、在受控端却显示未纳入”之类矛盾。本文先把这组关系固定下来，供两端共同遵循。

## 2. 三类终端集合

| 集合 | 含义 | 用途 |
|---|---|---|
| 教室受管集合 | 母机接管成功后当前管理的整批终端 | 查看整间教室、补录终端、导出清单、后续维护 |
| 本轮勾选终端 | 本次任务真正处理的子集 | 本轮部署 / 改 IP / 传桌面的作用对象 |
| 本轮保持现状终端 | 仍属本教室、本次不参与的终端 | 教师终端、待单独处理终端等可留在受管集合而不参与本轮 |

页面须能同时表达：本教室共有多少受管终端、本轮勾选了哪些、哪些保持现状。"保持现状"显示为"本次未纳入执行"，并回显其沿用的座位、机器名、IP、用途等当前内容，不显示为掉线或失败。

## 3. 母机视角与受控视角

- **母机视角**：整间教室有多少终端、本轮勾选多少、谁执行中、谁完成、谁失败。
- **受控视角**：本机是否被纳入、当前执行到哪一步、是否保持现状。
- 二者是同一轮任务的两个切面，须各自表达、口径一致；执行中的任务进度与结果停留在任务流程自身，不另设"收口态"中转。

## 4. 逐台确认层

批量任务会改写终端配置、桌面或启动设置，因此不在"设置完规则"后直接执行。标准次序：

1. 勾选终端并设置规则。
2. 把规则展开为逐台终端的写入结果（座位、IP、机器名、用途、是否纳入）。
3. 用户逐台核对或微调，处理例外项（排除教师终端、单机改值、局部跳过）。
4. 确认后才真正分发。

部署、维护、考试设置都遵循此层，区别只在展开的字段不同。逐台确认与微调以座位为单位进行：逐位呈现绑定与地址、逐位呈现传输进度与结果，并允许在结果态取消某座位、使下一轮跳过它。

## 5. 受控终端如何进入受管集合

受控终端被母机接管后转为受控态。在网络同传第二步，受控终端在各自屏幕上按回车 / 空格键，按布局方向顺序依次绑定到对应座位并同步该座位的 IP 与机器名；母机据此实时获知受管集合的绑定进度。绑定关系在进入下一步时固化。

## 6. 本地与服务器两条线

终端侧批量任务是否完成先看本地结果；终端状态、业务配置、桌面资产向服务器上传是另一条同步线，影响平台侧可见性与跨教室复用，但不反向推翻终端本地已完成的结果。详见 [云桌面终端与服务器按需同步说明.md](./云桌面终端与服务器按需同步说明.md)。
