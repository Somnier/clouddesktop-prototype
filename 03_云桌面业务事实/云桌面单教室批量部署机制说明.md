# 云桌面单教室批量部署机制说明

> 文档角色：业务事实层的终端结论之一。
> 作用：说明母机对整间教室批量部署的业务流程——各方角色、各方持有的数据、流程中数据如何变化，以及"受管集合 / 本轮勾选 / 保持现状"三类终端的关系。
> 前置：母机与受控的双模式概念见 [云桌面核心概念与统一口径.md](./云桌面核心概念与统一口径.md)；接管与教室绑定见 [云桌面母机接管与教室绑定机制说明.md](./云桌面母机接管与教室绑定机制说明.md)；数据上传见 [云桌面数据上传与一致性机制说明.md](./云桌面数据上传与一致性机制说明.md)。
>
> 事实来源：02_正式基线（初始需求清单、业务对象关系图、PRD）。

---

## 1. 各方角色与持有的数据

| 角色 | 持有 / 关注的数据 |
|---|---|
| 母机终端 | 整间教室的受管终端集合、本机桌面目录（部署来源）、布局与命名 / IP 规则、本轮任务的勾选与逐台写入计划 |
| 受控终端 | 本机身份（MAC）、本机被分配到的座位与写入的配置、本机当前任务状态 |
| 服务器（若在线） | 教室与终端登记信息、上传上来的状态 / 配置 / 桌面资产 |

## 2. 三类终端集合

| 集合 | 含义 |
|---|---|
| 教室受管集合 | 母机接管成功后当前管理的整批终端，是补录、导出、维护的总范围 |
| 本轮勾选终端 | 本次任务真正处理的子集 |
| 本轮保持现状终端 | 仍属本教室、本次不参与的终端（如教师终端、待单独处理终端） |

"保持现状"表示仍在受管集合内、本次不执行，沿用其既有座位 / 机器名 / IP / 用途，不是掉线或失败。

## 3. 业务流程与数据变化

批量部署是一条连续主流程，依次经历布局、绑定与地址、桌面传输三个阶段；各阶段的数据变化如下。

### 3.1 布局阶段

- 母机设定教室名称、网格行列与布局方向、座位起始字母、机器名前缀、IP 规则。
- 后台开始发现局域网内的终端，隐式维护当前教室的接管终端列表。
- 此阶段产生的是"布局与规则"数据，尚未写入任何受控终端。
- 座位是否启用、是否存在由布局决定：教室物理布局未必规整，某些位置可能没有终端或被占用，因此布局允许把个别座位标记为不可用，被排除的座位不参与编号与 IP 分配；也允许保留一个空位占用编号 / IP 名额，供后续补装设备。

### 3.2 绑定与地址阶段

- 母机接管使受控终端进入受控态。
- 受控终端按物理位置依次主动确认，绑定到布局中的对应座位；绑定后该终端获得该座位对应的座位号、机器名、IP 等配置。
- 绑定顺序遵循布局方向；母机据此实时获知受管集合的绑定进度。
- 用户进入下一阶段时，绑定关系与地址分配固化。
- 此阶段把"布局规则"展开为逐台终端的具体写入值。

### 3.3 桌面传输阶段

- 母机把所选桌面（可多选，默认启动桌面从已选中再选 1 个）连同启动设置，分发到本轮勾选的受控终端。
- 传输过程中各受控终端依次经历传输 / 写入 / 重启；完成后本机配置与桌面落地。
- 本地传输完成即算该终端部署完成，与是否已上传服务器无关。

## 4. 逐台确认

批量任务会改写终端的配置、桌面或启动设置，因此不在"设置完规则"后直接全量执行，而是把规则展开为逐台写入结果，供用户核对、微调、处理例外项（排除教师终端、个别改值、局部跳过），确认后才分发。部署、维护、考试设置都遵循逐台确认，区别只在展开的字段不同。

## 5. 母机视角与受控视角

- **母机视角**：整间教室有多少终端、本轮勾选多少、谁绑定 / 执行中 / 完成 / 失败。
- **受控视角**：本机是否被纳入、绑定到哪个座位、执行到哪一步、是否保持现状。
- 二者是同一轮任务的两个切面，须各自表达、口径一致。任务执行期间受控终端的各状态反映见 [云桌面任务执行中断与受控终端任务屏机制说明.md](./云桌面任务执行中断与受控终端任务屏机制说明.md)。

## 6. 与服务器的关系

批量部署的完成判断只看本地结果。终端状态、业务配置、桌面资产向服务器上传是另一条线，影响平台侧可见性与跨教室复用，但不反向推翻本地已完成的结果，详见 [云桌面数据上传与一致性机制说明.md](./云桌面数据上传与一致性机制说明.md)。
