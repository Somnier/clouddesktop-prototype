## 1. 云桌面在做什么

云桌面是一套**通用终端管理方案**：以终端本地运行能力为基础、以服务器统一纳管能力为增强，管理多校区、多教室、多终端的桌面（操作系统环境）生命周期。它不是重型 VDI——桌面运行在终端本机，不依赖服务器持续供给算力。

核心能力链：在教室里把一台终端的桌面与配置，批量复制到整间教室的其它终端；再把教室的终端清单与桌面上报到服务器，供跨教室复用与统一运维。

---

## 2. 双端：终端侧与平台侧

系统由两端组成，两端共享同一批业务对象，但视角与职责不同。

### 2.1 终端侧

- 指运行在**教室终端**上的管理软件（深色工控风界面）。
- 面向现场工程师/机房管理员，完成本机配置、桌面制作、整间教室的批量部署与维护。
- 可在**无服务器**的情况下独立完成单教室交付。

### 2.2 平台侧（服务器侧）

- 指运行在**服务器**上的 Web 管理平台（浅色数据管理风界面）。
- 面向管理员，统一查看与纳管多校区、多教室、多终端，复用桌面资产。
- **“平台侧”与“服务器侧”是同一端的两种说法**：“平台 / 平台侧”是口语与架构层面的称呼；界面中凡涉及服务器的动作与字段一律使用“服务器”（如“服务器地址”“同步到服务器”“上传到服务器”）。

### 2.3 两端关系与数据一致性

- 终端侧本地交付不以服务器为前提；服务器是统一纳管与跨教室复用的增强。
- 全系统共享同一份数据：终端侧上报的数据是平台侧数据的唯一来源，平台侧不凭空产生终端数据。
- 因此一旦某教室的终端清单导入服务器，该教室每一台终端的后续改动（机器名、IP、用途、桌面、产生的日志等）都会同步反映到平台侧；平台呈现的教室数、终端数、桌面数也据此与终端侧汇总一致。同步范围与边界见[终端与服务器按需同步说明](./云桌面终端与服务器按需同步说明.md)。

## 3. 终端的两种模式：母机终端与受控终端

母机与受控**不是固定的设备类型，而是终端在某一时刻承担的工作模式**。任一终端默认都是"本机终端"，只有在网络同传场景下才分化出母机与受控两种角色。

### 3.1 本机终端（默认态）

- 任一终端未参与网络同传时，就是一台普通的"本机终端"，只管理自己：设置本机信息、管理本机桌面、导出本机清单。

### 3.2 母机终端

- 在网络同传中承担"管理整间教室"角色的那一台终端，即母机终端。
- **未使用网络同传（教室接管）功能时，母机终端表现为普通本机终端**——它没有特殊设备身份，只是当前恰好承担了母机角色。
- 母机是教室桌面与配置的来源方/上传方：它持有整间教室的完整桌面目录，并据此向其它终端分发、向服务器上传。

### 3.3 受控终端

- 在网络同传中被母机接管、进入受控状态的终端。
- **未被接管时，受控终端同样表现为普通本机终端**。
- 受控期间，受控终端不可自行发起单机操作（如重置/替换/改本机设置），以免打断控制关系。

### 3.4 教师终端与学生终端

- 这是按用途对终端的标记，与母机/受控模式正交：教师终端通常位于讲台（座位 T01），持有全部桌面；学生终端按需获得面向学生的桌面。
- 母机角色通常由教师终端承担，但二者概念独立——母机是"管理角色"，教师终端是"用途标记"。

## 4. 概念—口语—界面文字映射

业务概念、现场口语、与界面上呈现的文字可以不同，但须一一对应、不半改半留。文档解释机制时用左列概念，描述界面时用右列文字。

| 业务概念 | 现场口语 | 界面文字 |
|---|---|---|
| 教室接管 / 母机接管 | 接管教室、控制整个机房 | 网络同传 |
| 母机工作模式 | 当主控、当老师机 | 网络同传中的母机角色 |
| 平台侧 / 服务器侧 | 传到平台、传到服务器 | 服务器（服务器地址、同步到服务器、上传到服务器） |
| 设置本机 + 设置服务器 | 配一下这台机器 | 设置本机（合并为一页） |
| 单教室批量部署 | 装机、批量装系统、推系统 | 网络同传 → 传输桌面 |
| 桌面（操作系统环境） | 系统、镜像、还原点 | 桌面；离线包为桌面包（.cdpkg） |
| 受控终端绑定座位 | 学生机报到、认座位 | 受控终端按回车 / 空格键绑定座位 |
| 教室维护（批量改 IP / 传桌面） | 改地址、重装 | 网络同传内的 IP 设置 / 传输桌面 |
| 导出教室终端清单 | 导名单、备份配置 | 导出终端清单 |
| 终端业务配置 | 机器设置 | 设置本机各字段（座位 / 机器名 / IP 等） |

“教室接管 / 母机工作模式 / 单教室部署 / 教室维护”这几条概念，在界面上收敛为“网络同传”一条主流程。概念层文档继续使用左列业务术语，界面层一律以右列文字为准。

## 5. 桌面、镜像、快照的概念位置

- **桌面**：用户与界面打交道的对象，即一个可启动的操作系统环境（含已装软件、配置、数据盘）。
- **基础镜像 / 快照**：桌面底层依赖的技术层数据，按需随桌面同步。界面主路径只呈现“桌面”，技术层概念不向普通用户暴露。
- 三者的数据模型关系见[镜像快照桌面三层资产模型说明](./云桌面镜像快照桌面三层资产模型说明.md)。

## 6. 数据分层口径（速查）

页面数据至少分五层，不可用一条字段混表多层含义：

1. **当前实时状态**：当前在线、当前机器名、当前 IP、当前工作 MAC、当前连通性。
2. **终端业务配置**：教室、座位号、终端用途、机器名、网络配置、服务器地址、默认启动项、还原方式。
3. **当前任务数据**：本轮勾选了哪些终端、顺序、准备写入什么。
4. **当前任务运行状态**：当前步骤、进度、失败原因、同步状态。
5. **服务器侧/平台侧汇总数据**：终端登记信息、桌面资产状态、平台汇总。

字段口径细则见[页面表达与字段口径说明](./云桌面页面表达与字段口径说明.md)。
