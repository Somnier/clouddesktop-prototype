# 07 可行性与工程筹备

> 文档角色：从「事实」到「原型」之间的过渡层。
> 作用：沉淀原型落地前的可行性论证、原型范围取舍，以及面向后续同事 / Agent 的交接精要。本层不定义业务事实或设计事实（那些在 03 / 04），也不是可运行原型（在 08）；它承接"为什么这样做、做到哪、怎样接手"。通用工程方法论已移至 [09_方法指南](../09_方法指南/README.md)。

## 内容分组

### 交接精要
- [云桌面项目交接精要.md](./云桌面项目交接精要.md)：目录分层与阅读顺序、默认事实锚点、已稳定的产品边界、写法偏好与禁忌、原型技术栈速览。**接手者应最先阅读。**

### 原型范围取舍
- [云桌面原型范围取舍.md](./云桌面原型范围取舍.md)：原型做什么、不做什么，以及当前未实现的能力清单。

### 可行性分析
- [可行性分析/](./可行性分析/README.md)：原型落地前的技术背景判断、研发可行性与工程落地路径、纯前端原型展示可行性论证。

## 与其它层的衔接

1. 上游：`02_正式基线`、`03_云桌面业务事实`、`04_云桌面设计事实` 是本层判断所服从的前提。
2. 下游：`08_云桌面原型` 是本层可行性与范围取舍的落地实现。
3. 旁路：`09_方法指南` 沉淀通用工程方法论，供关注协作节奏与文档写法时参考。
4. 本层材料服务于接续与复用，不反向变成正式事实前提。
