# 云桌面原型范围取舍

> 文档角色：设计事实层的核心文件之一。
> 作用：界定原型"做什么、不做什么"。页面设计（终端侧、平台侧）保留完整设计，原型按本文取舍实现；设计描述了而原型未实现的能力，以本文的"不做"清单为准，属正常的范围取舍，不视为缺陷。

---

## 1. 做什么（核心范围）

- **可视化教室管理**：用二维座位网格映射物理座位，替代 MAC / 列表抽象。
- **集中的主流程**：把"扫描发现 → 命名 / IP 规则 → 座位绑定 → 桌面传输"收敛进一条可自由切换的网络同传流程，减少页面跳转与上下文丢失。
- **故障即时恢复**：终端首页一级入口提供"替换故障终端""重置终端"，应对现场应急。
- **平台单屏概览**：Web 侧以三栏布局提供仪表盘（健康度、服务器监控、告警日志）的单一视图。
- **两端数据一致**：座位 / 机器名 / IP 在两端按统一口径与顺序呈现；桌面资产由终端上报汇总，平台侧据共享数据呈现。
- **导出—导入闭环**：终端导出（单机或整教室）→ 服务器按教室名归并、按清单自动导入桌面包（机制见业务事实 [终端导出与平台导入机制说明](../03_云桌面业务事实/云桌面终端导出与平台导入机制说明.md)）。

## 2. 不做什么（当前未实现）

原型聚焦核心运维，以下能力**当前不实现独立页面与流程**：

| 未实现项 | 现状 |
|---|---|
| 考试流程 | 终端首页无考试入口，无对应页面。考试桌面仅作为一种桌面数据类型存在；其完整设计保留在终端侧页面设计的考试桌面流程文档，供后续实现参考。 |
| 终端自检流程 | 首页与导航均无入口。 |
| 资产同步进度详情 | 不设独立进度页；改为后台自动同步 + 桌面卡片上的状态标识（未同步 / 同步中 / 已同步）表达。 |
| 独立的多步部署向导 | 不设独立向导；部署收敛进网络同传第三步，三步可自由切换、就地展示参数 / 进度 / 结果。 |

## 3. 设计与实现的对齐

1. 页面设计文档（终端侧 / 平台侧）是完整设计的载体，包含已实现与当前未实现的页面。
2. 原型是可运行实现，其覆盖范围以本文第 1、2 节为准。
3. 当页面设计描述的某流程在原型中未实现时，以本文"不做"清单为准，不应被误判为实现缺失。
4. 当某能力进入实现排期时：先在页面设计补齐 / 确认设计 → 在原型实现 → 回到本文更新清单。
