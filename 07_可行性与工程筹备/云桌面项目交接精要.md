## 3. 已稳定的产品边界

1. 本产品不是重型 VDI，而是"终端本地运行为基础、服务器统一纳管为增强"的云桌面方案。
2. 终端侧核心价值：脱机单教室部署（母机批量 + 单终端自行）、离线桌面包携带跨教室复用、日常维护友好、一键替换与一键重置。
3. 平台侧核心价值：自动发现告警、批量测试、桌面资产管理（导入导出与删除无引用桌面）。
4. 多教室交付的基础路径仍是重复单教室部署；服务器复用与离线打包是并列增强。
5. 线下角色（安装工程师、机房管理员、技服工程师、教师）与平台账号（系统管理员、校区子管理员）分开建模；账号与授权管理为二期概念。
6. 教师终端不是独立硬件，而是普通终端加教师用途标记与教师桌面能力。
7. 用户侧统一用"桌面"，不暴露"镜像 / 快照"；桌面包是离线携带单元，内部三层细节不暴露。
8. 教室终端清单含全部终端信息与桌面包，导入服务器时一并导入。
9. MAC 是内部关键标识必须保留，但主路径弱化展示、以座位号 / 机器名替代，详情与诊断场景再展示完整 MAC。
10. 测试与告警是产品内能力；一键重置当前依赖服务器作为权威恢复源；服务器地址变更是低频复杂的独立操作。

## 4. 写法偏好与禁忌

1. 正式正文只写当前成立的结果，不写修订日志、补丁记录、历史回顾。
2. 不把每次纠偏内容制度化成一堆新重点；改掉错误、收束口径即可。
3. 不制造标题 / 正文 / 按钮各用一套术语的半替换口径。
4. 当前值、已保存配置值、本次准备写入值必须分开表达，不混成一条字段。
5. 同一条规则若影响多个页面，先修事实层再修页面层；小改动只按小改动处理。
6. 页面动作的摆放位置本身就是语义，不把真实操作按钮藏进说明文字。
7. 事实不清时先回 `02` 基线、`03/04` 事实与当前页面稿，不先发明新清单或新对象。

## 5. 原型技术栈速览

Node.js + Express + SSE（约 600ms tick），端口 3920；种子数据在 `08_云桌面原型/data/seed.json`，运行时状态由 `server.mjs` 的 `buildState()` 从种子生成。四端（导演台 / 母机 / 受控 / 平台）共享同一份状态。可行性与技术背景详见本层 [可行性分析](./可行性分析/README.md)。
