# 平台侧页面信息架构

> 文档角色：07 目录的总纲。
> 作用：定义平台侧页面树、导航结构、跨页原则和通用布局约束。各 WEB 页面文档负责展开各自正文。

## 1. 页面编号与职责

| 编号 | 页面名称 | 职责 |
| :--- | :--- | :--- |
| WEB-01 | 平台总览 | 校区级运行概览、服务器状态、任务进度、异常与风险、教室概况 |
| WEB-02 | 教室管理 | 教室列表、导入终端清单、教室详情（4 Tab）、单教室与跨教室批量操作 |
| WEB-02A | 终端详情 | 从 WEB-02 进入的单终端全量信息查看 |
| WEB-03 | 桌面资产 | 校区级桌面资产管理，支持桌面包导入导出和清理无引用桌面 |
| WEB-04 | 告警中心 | 校区级告警汇总、按教室/类型筛选与排序、跳转教室和终端 |
| WEB-05 | 系统运维 | 授权管理、服务器地址变更、校区与账号管理 |

## 2. 导航结构

```text
┌──────────────────────────────────────────────────────────────┐
│ 侧边栏                         │ 主内容区                    │
│                                 │                             │
│ 云桌面管理平台                  │ 面包屑：首页 / 教室管理 / …  │
│ ────────────                    │ ──────────────────────────── │
│ 导航                            │                             │
│   平台总览                      │  （当前页面内容）            │
│   教室管理                      │                             │
│   桌面资产                      │                             │
│   告警中心                      │                             │
│   系统运维                      │                             │
│ ────────────                    │                             │
│ 校区                            │                             │
│   杨浦校区  ●                   │                             │
│   宝山校区                      │                             │
└──────────────────────────────────────────────────────────────┘
```

导航规则：

1. 侧边栏始终可见，包含 5 个一级页面入口和校区切换。
2. 切换校区后，当前页面自动刷新为目标校区数据，不回到首页。
3. 面包屑按页面层级展示：首页 → 教室管理 → 教室名称 → 终端名称。
4. 从任意告警、日志或指标卡片上的教室名、终端名可点击跳转到对应详情页。

## 3. 平台侧三类视角

平台侧的所有页面可以归为三类视角：

1. **汇总视角**（WEB-01、WEB-04）：以校区为粒度汇总查看，不修改数据。
2. **管理视角**（WEB-02、WEB-02A、WEB-03）：以教室、终端或桌面为粒度管理资产和执行批量操作。
3. **运维视角**（WEB-05）：服务器和平台本身的管理与配置。

```text

  ├─ 服务器状态
  ├─ 任务进度
  ├─ 异常与风险
  ├─ 教室概况（点击 → WEB-02 教室详情）
  └─ 最近日志

WEB-02 教室管理
  ├─ 教室列表
  │   ├─ 导入终端清单（向导，含桌面包导入进度）
  │   └─ 教室行（点击 → 教室详情）
  └─ 教室详情
      ├─ Tab：总览
      ├─ Tab：终端列表（含批量工具栏）
      │   ├─ 座位图视图 / 列表视图
      │   ├─ 单教室操作：批量关机 / 批量重启 / 单教室桌面部署（3 步向导）/ 批量修改 IP
      │   ├─ 单教室测试：网络连通性测试
      │   ├─ 多教室操作：跨教室桌面部署（3 步向导，目标可选其他教室任意终端）
      │   └─ 多教室测试：跨教室广播隔离测试
      ├─ Tab：桌面与存储（只读，细化到终端级使用明细）
      └─ Tab：告警

WEB-02A 终端详情（从 WEB-02 终端列表进入）
  ├─ 配置信息
  ├─ 外部连接
  ├─ 硬件信息
  ├─ 运行状态
  ├─ 已部署桌面列表
  ├─ 当前任务
  └─ 告警

WEB-03 桌面资产
  ├─ 校区级指标
  ├─ 桌面列表（以桌面为第一层级，标注使用教室与终端数）
  ├─ 镜像与快照链（只读内部资产视图）
  └─ 桌面包操作（导入 / 导出 / 删除无引用桌面）

WEB-04 告警中心
  ├─ 严重度摘要
  ├─ 筛选（教室 / 类型）与排序（时间 / 严重度）
  └─ 告警列表（跳转教室 / 终端）

WEB-05 系统运维
  ├─ 授权管理
  ├─ 服务器地址变更
  └─ 校区与账号管理
```

## 5. 跨页原则

### 5.1 数据分层

平台侧页面至少要区分以下数据层，不能混为一条字段：

1. 服务器端登记信息：教室、座位、终端用途、MAC 地址列表、当前工作 MAC。
2. 终端上报的运行状态：在线状态、CPU/内存负载、最近心跳时间。
3. 终端上报的桌面资产：桌面目录、快照链、数据盘。
4. 任务进度：当前执行态、阶段、已完成/失败数量。
5. 测试结果与告警记录。

详见 [云桌面跨端结论_页面表达与字段口径说明.md](../05_云桌面事实/云桌面跨端结论_页面表达与字段口径说明.md)。

### 5.2 平台侧不重写终端侧判断

平台可以展示终端侧的本地结果，但不应反向改写终端侧的完成判断。详见 [云桌面平台侧结论_页面范围与任务边界说明.md](../05_云桌面事实/云桌面平台侧结论_页面范围与任务边界说明.md)。

### 5.3 在线状态不等于执行态

"在线"只代表终端当前网络可达。教室列表、教室页和终端列表都应把在线状态、当前执行态、任务摘要、异常摘要和测试摘要拆开表达。

### 5.4 MAC 弱化原则

MAC 地址是系统内部关键标识，但 UI 主路径优先用座位号和机器名标识终端。终端详情页和诊断场景保留完整 MAC 信息。

### 5.5 桌面资产管理原则

平台侧不直接制作或编辑桌面内容。桌面资产数据来自教室终端清单导入和终端自行上报。平台可发起桌面下发，但不修改桌面内容本身。平台支持桌面包的导入、导出和删除无引用桌面（PRD FR-04、初始需求清单 §1）。镜像和快照作为只读内部资产视图展示，不可单独操作。

## 6. 通用布局约束

1. 平台侧为 Web 应用，运行于桌面浏览器，目标分辨率 1920×1080。
2. 侧边栏固定宽度，主内容区自适应。
3. 列表页支持分页或滚动加载。
4. 指标卡片使用网格布局，在窄屏时自动换行。
5. 批量操作面板展开时，不遮挡终端列表或座位图，两者联动显示选中状态和进度。

## 7. 主线流程路径

### 7.1 首次交付后平台查看

WEB-01 平台总览 → WEB-02 教室列表 → 导入终端清单 → 教室详情 → 终端列表核对

### 7.2 日常巡检

WEB-01 平台总览 → 查看异常教室 → WEB-02 教室详情 → WEB-04 告警中心

### 7.3 跨教室桌面部署

WEB-02 教室详情 → 终端列表 → 跨教室桌面部署 → 选来源终端与桌面 → 选目标终端 → 预览 → 执行

### 7.4 服务器地址变更

WEB-05 系统运维 → 输入新地址 → 通知在线终端 → 查看未送达清单 → 终端侧后续处理
