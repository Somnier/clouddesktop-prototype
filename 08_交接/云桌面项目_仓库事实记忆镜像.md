## 3. 跨端稳定结论

1. 页面数据至少要分成当前实时状态、终端业务配置、当前任务数据、当前任务运行状态、服务器侧或平台侧数据 5 层。
2. 当前值、配置值和这次要写入的值必须分开表达，不能混成一条字段。
3. 要同时定义母机页、受控任务屏和 TERM-06 的边界，不是因为它们属于同一条执行中的流程，而是为了明确互斥关系：母机页和受控任务屏属于同一轮批量任务的两个视角；TERM-06 只在终端未受控、任务已中断、任务已取消，或终端已经明确脱离整批任务后才成立。
4. 术语层继续稳定使用“母机”“受控终端”“教师终端”“学生终端”；不要再做标题和正文两套口径的半替换。
5. TERM-04、TERM-05、TERM-07 的待下发桌面集合都可以多选，但来源只能是当前母机已经准备好的本机桌面；复制桌面和进入桌面编辑属于 TERM-01 的本机桌面动作，不再出现在批量任务页；批量部署、批量维护、考试前设置和考后恢复处理的是一批终端。机器名、座位号、终端用途、网络配置、服务器地址、桌面集合、默认启动项和还原方式，在单机场景下由 TERM-01 的对应本机页逐项处理，含义不另起一套。进入桌面编辑时，终端会重启进入当前选中桌面的桌面环境；完成这次桌面处理并再次重启后，系统回到云桌面管理系统，并定位回发起它的页面。
6. TERM-04 不应假设部署前已经存在可靠的教师终端或学生终端快捷筛选；更稳妥的是提供全选、字段排序和逐台勾选，再把终端用途作为部署写入的一部分处理。
7. 当前教室受管集合、这轮勾选终端、保持现状终端必须拆开表达。
8. 批量任务都应该有逐台确认层，部署、维护和考试都沿用这条规则。
9. 本地完成和服务器同步是两条并行线；服务器同步失败不反向推翻终端本地已完成结果。
10. 只要列表需要人工勾选终端、桌面、教室或其他对象，就应支持排序和分页；可排序字段、默认排序字段和分页策略按页面对象和数据量确定，不统一写死。
11. “平台侧”用于 Web 页面与平台用户操作视角；“服务器侧”用于服务器本体动作、服务器端登记信息和服务器侧记录的数据；“配置”用于持久业务值；“设置”用于用户当前正在选择、确认或准备写入的动作条件与任务参数。

## 4. 当前终端侧页面结论

1. TERM-01 到 TERM-07 由终端总纲管理跨页关系，各 TERM 文档负责页面正文。终端总纲新增§4"终端侧通用布局原则"，约束所有终端侧页面必须在 1920×1080 下一页完成、不出现页面级滚动条、步骤条和操作按钮始终可见、表单间距紧凑。
2. TERM-01 只回显终端用途，不承担独立设置；首页里的“当前管理态”只用于重进软件后的恢复入口。
3. TERM-02、TERM-03 负责母机接管和教室工作面；TERM-02 负责接管识别与归属判断，TERM-03 负责接管成功后的当前教室工作面与教室级入口；接管和教室绑定边界由 `05_云桌面事实/云桌面母机接管与教室绑定机制说明.md` 约束。
4. TERM-04、TERM-05、TERM-07 共享同一纯批量骨架：当前母机先准备本机桌面，再选这轮终端、选择已有目标桌面、逐台确认、批量执行、结果停留；TERM-04 的 [修改本机桌面] 放在当前母机准备区，TERM-05、TERM-07 的 [修改本机桌面] 放在各自桌面选择步骤；桌面处理完成并再次重启后，系统回到原页面的原步骤；同一组业务字段在单机场景下由 TERM-01 的本机页逐项处理。
5. TERM-06 负责脱离整批任务后的单终端替换、重置和网络修正；如果服务器不可达，必须在当前流程内联复用“网络与服务器”字段组。
6. TERM-01 受控任务屏区分 8 种画面状态：等待任务开始、执行中、已完成、执行失败、本次未纳入执行（保持现状）、任务中断、母机连接丢失、受控空闲。

### 4.1 原型评审确认的页面结构细节

1. 终端用途字段在空白教室默认为"未设定"，而不是 null 或空字符串。原型中终端用途下拉选项为"未设定""教师终端""学生终端"。
2. TERM-01 首页结构为四栏并排信息区（可配置项 / 外部连接 / 硬件信息 / 运行状态）+ 两组按钮（单机功能 / 教室管理）。可配置项按身份→网络→桌面顺序：机器名、座位号、终端用途、IP 地址、默认启动桌面，不包含还原模式。硬件信息优先级：网卡列表、处理器、显卡、内存、硬盘。运行状态（CPU/GPU/内存/磁盘 4 指标）与其他三栏等宽并排，不再单独占宽行。外部连接包含服务器地址、服务器状态、终端注册状态、最近同步时间。本机配置页不再包含 MAC 地址列表（已在导航栏展示，不可编辑）。整页在 1920×1080 分辨率下不出现页面滚动条。
3. 桌面管理页（本机桌面）顶部为磁盘空间摘要条（SVG 饼图 + 桌面/系统/剩余三色占比），不再有 BIOS 引导策略独立卡片。下方为导入/导出按钮和桌面卡片列表。
4. 每张桌面卡片包含：桌面名称（无自动版本号）、默认启动标记、可见性标记（"显示"/"已隐藏"）、上传状态（"已上传"/"未上传"）、物理部署标记、操作系统、基础镜像名称、更新时间、备注、数据盘列表（盘符/大小/共享状态）、还原模式下拉（每个桌面独立设置）、操作按钮（进入编辑 / 复制 / 设为默认 / 隐藏 / 删除）。
5. 删除默认启动桌面时，系统自动将默认启动顺延到下一个桌面，而不是禁止删除。删除未上传桌面时需额外警告不可恢复。
6. 受控终端页面是纯任务屏，不包含本地导航和本机操作功能。受控终端区分 8 种画面状态：等待任务、执行中、已完成、失败、保持现状、任务中断、母机连接丢失、受控空闲。
7. 母机终端 = 普通终端的全部单机功能 + 教室管理模式。进入教室管理需要先接管教室。
8. 导航栏身份信息格式：在线状态圆点 + "在线"/"离线" 文字 + MAC 地址 + 机器名 + 座位号 + IP 地址，各项之间用分隔符或背景色卡片隔开。
9. 导入桌面区分两条路径：基础镜像文件（.vhd/.vhdx/.iso 等）需经过创建桌面流程（配置数据盘、部署方式、还原模式），成品桌面包（.ljpkg）直接生成桌面跳过创建流程。
10. 数据盘每桌面独立，默认不共享。新建桌面时可选择复用已有数据盘或新建隔离数据盘，盘符默认 D: 顺延，但要考虑不同文件系统使用不同方案表达“盘符”概念。
11. 一键重置前置条件为"服务器可达 + 终端已注册到服务器"。
12. 本机测试页包含：网络连通性 → 硬盘读写速度 → 硬盘健康状态，每项支持单独重新测试。

## 5. 母机接管与任务机制结论

1. 母机接管要区分初次接管和后续接管；后续接管优先扫描终端本地保存的教室信息，而不是只看当前母机本机状态。
2. 终端侧不应补出正式的“创建教室”流程，更适合围绕当前教室终端集合工作。
3. 初次接管默认尽量纳入当前发现到的全部可管理终端；后续接管支持补录并入遗漏终端。
4. 当前阶段按单母机占用处理；主释放路径是母机主动退出管理，异常脱管处理不在页面阶段写死细节。
5. 批量任务必须支持母机侧主动取消；已完成终端保持结果，未完成终端不按成功处理。

## 6. 系统与数据流转结论

1. 本产品不是重型 VDI，而是“终端本地运行为基础、服务器统一纳管为增强”的云桌面教学方案。
2. 多教室交付的基础路径是重复单教室部署；服务器辅助复用和离线打包是增强路径。
3. 教师终端不是独立硬件体系，而是普通终端加教师用途标记和教师桌面能力。
4. 教室终端清单可以先导出并导入服务器创建教室，再继续制作桌面；系统必须兼容"先创建教室、后制桌面"。
5. 多网卡场景必须保留 MAC 地址列表和当前工作 MAC；MAC 地址是系统内部的关键标识，但用户阅读 MAC 困难，因此 UI 主路径应尽量弱化展示，转而用座位号、机器名等业务概念替代；详情页和诊断场景仍应保留完整 MAC 信息。
6. 一键重置当前依赖服务器作为权威恢复源；本地已有镜像、桌面和快照属于本机运行数据，但不作为可信恢复源。
7. 终端业务配置、终端当前状态和服务器端登记信息需要分层表达，不能做成一张万能信息表。
8. 服务器地址变更后，对漏通知或离线终端的后续处理应允许直接前往目标终端单独修改网络与服务器配置，不要求必须先经过其他批量入口。

## 7. 平台侧结论

1. 平台侧共 5 个主页面和 1 个详情子页：WEB-01（平台总览）、WEB-02（教室管理）、WEB-02A（终端详情）、WEB-03（桌面资产）、WEB-04（告警中心）、WEB-05（系统运维）。
2. WEB-02 同时承接单教室和跨教室批量操作，工具栏分 4 组：单教室桌面部署、单教室测试、跨教室桌面部署、跨教室广播隔离测试。
3. 平台侧只能看见服务器已纳管的终端信息，不能反向改写终端本地已完成结果。
4. 平台侧的桌面资产管理（WEB-03）支持桌面包导入导出和无引用桌面清理，但资产修改不直接推给终端。
5. 告警中心（WEB-04）按教室和类型筛选排序，支持跳转教室和终端详情。
6. 系统运维（WEB-05）承接授权管理、服务器地址变更、校区与账号管理；服务器地址变更后应生成未送达终端清单并给出终端侧后续处理入口。

## 8. 考试、维护与交付结论

1. 考试、部署、维护三类教室级任务都应共享“可取消但不显式承诺回退细节”的页面口径。
2. 考试场景当前更适合表达为“考试启动设置与考后恢复”，恢复依据是考前启动设置快照。
3. 数据盘大小、复用已有数据盘、新建独立数据盘等选择已回收到通用桌面准备逻辑，不再只挂在考试专项下。
4. 交付测试属于产品内能力；临时母机当前教室界面和服务器平台都可发起、查看和导出常用测试。

## 10. 原型评审确认记录

> 以下内容在原型实现与评审迭代中被确认，作为补充事实记录。

1. 终端用途字段在空白教室默认为"未设定"，不是空值。下拉选项固定为"未设定""教师终端""学生终端"。
2. 受控终端任务屏实际区分 8 种画面状态（在原文档"6 种"基础上增加"等待任务开始"和"母机连接丢失"）。
3. 删除默认启动桌面时，系统自动将默认启动顺延到下一个可用桌面，而不是禁止删除操作。
4. 桌面可见性标记使用"显示"和"已隐藏"两个中文标签，不使用英文标记。
5. TERM-01 首页结构已确认为四栏并排信息区（可配置项 / 外部连接 / 硬件信息 / 运行状态）+ 两组操作按钮（单机功能 / 教室管理）。可配置项不包含还原模式，新增 IP 地址字段。硬件信息优先展示网卡列表。运行状态（CPU/GPU/内存/磁盘）与其他三栏等宽并排。
6. 桌面管理页顶部为磁盘空间摘要条（SVG 饼图 + 桌面/系统/剩余三色占比），不再使用 BIOS 引导策略独立卡片。还原模式改为每个桌面独立设置。
7. 导航栏结构：在线状态标识（圆点 + "在线"/"离线" 文字）+ 身份四项（MAC · 机器名 · 座位号 · IP），各项之间使用视觉分隔符。
8. 危险操作（如重置数据、删除桌面）使用自定义确认弹窗，不使用浏览器原生 confirm/alert 对话框。删除未上传桌面时需额外警告不可恢复。操作系统文件选择框可以使用原生实现。
9. 导入桌面区分基础镜像（需创建桌面流程）和成品桌面包（直接生成），使用选择框让用户先选导入类型。
10. 桌面卡片新增字段：上传状态（已上传/未上传）、物理部署标记、基础镜像名称、数据盘列表（盘符/大小/共享）、每桌面独立还原模式下拉。
11. 首页外部连接卡片包含：服务器地址、服务器连接状态、终端注册状态、最近同步时间。
12. 所有页面在 1920×1080 分辨率下不出现页面滚动条；如列表或表格溢出，只允许该区域内部滚动，不将按钮藏在页面滚动条底部。
13. 表单标签与输入框间距收紧（gap: 12px），不再使用 space-between 拉满整行。
14. TERM-02 接管页扫描终端列表只显示在线终端，移除"在线"列；新增"归属"列显示终端本地记录的教室名称；如存在其他教室终端（VLAN 问题）则分组显示。
15. TERM-03 工作页告警区移至页面顶部卡片位置；教室信息卡简化为统计数据（在线/离线、已部署/未部署），不含母机行；移除"桌面管理"按钮；"结束本次管理"改为"结束管理"。
