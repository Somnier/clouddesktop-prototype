# 云桌面 UI 呈现原则

> 文档角色：设计事实层的核心文件之一。
> 作用：沉淀云桌面在「视觉与排版呈现」上的统一规则，约束所有页面的呈现方式。
> 适用范围：终端侧（深色工控风）与平台侧（浅色数据管理风）双端。两端共用同一套抽象原则，仅在具体色板（设计令牌）上区分明暗主题。

---

## 1. 设计令牌（颜色与基准）

颜色一律通过 CSS 变量（设计令牌）引用，严禁在页面中硬编码十六进制色值；新增颜色需求应先回到本节登记令牌。

### 1.1 平台侧（浅色主题，`base.css :root`）

| 用途 | 令牌 | 色值 |
|---|---|---|
| 背景 / 卡片 / 边框 | `--c-bg` / `--c-surface` / `--c-border` | `#f5f6f8` / `#fff` / `#e2e5ea` |
| 主文字 / 次文字 / 弱文字 | `--c-text` / `--c-text2` / `--c-text3` | `#1a1d23` / `#5a6170` / `#8b92a0` |
| 品牌主色（操作 / 链接 / 选中） | `--c-brand` | `#2563eb` |
| 成功 / 在线 | `--c-ok` / `--c-ok-bg` | `#059669` / `#d1fae5` |
| 警告 | `--c-warn` / `--c-warn-bg` | `#d97706` / `#fef3c7` |
| 错误 / 严重 | `--c-err` / `--c-err-bg` | `#dc2626` / `#fee2e2` |

### 1.2 终端侧（深色主题，`terminal.css .term-app`）

| 用途 | 令牌 | 色值 |
|---|---|---|
| 背景 / 面板 / 边框 | `--t-bg` / `--t-panel` / `--t-border` | `#0e1117` / `#161b22` / `#30363d` |
| 主文字 / 次文字 / 弱文字 | `--t-text` / `--t-text2` / `--t-text3` | `#e6edf3` / `#9da5b0` / `#6e7681` |
| 强调主色（操作 / 链接 / 选中） | `--t-accent` | `#58a6ff` |
| 成功 / 在线 | `--t-ok` / `--t-ok-bg` | `#3fb950` / `rgba(63,185,80,.12)` |
| 警告 | `--t-warn` / `--t-warn-bg` | `#d29922` / `rgba(210,153,34,.12)` |
| 错误 / 严重 / 离线 | `--t-err` / `--t-err-bg` | `#f85149` / `rgba(248,81,73,.12)` |

### 1.3 语义色使用约定

颜色承载语义，不可混用：

- 品牌 / 强调蓝：仅用于「可操作」与「选中 / 聚焦」状态——主按钮、链接、当前页签、选中座位。不用蓝色去“强调”一个非操作状态（例如“默认启动”标签用中性灰 `muted`，而非蓝框）。
- 绿（ok）：成功、在线、已完成、已绑定。
- 黄（warn）：需用户注意但尚未出错的警告态——未同步、待处理、容量偏高，以及与警告直接相关的行动按钮（如针对未同步态的“同步到服务器”按钮）。
- 红（err）：错误、严重告警、失败、离线（硬件层面不可用）、危险操作（删除 / 重置）。
- 中性灰（muted / text3）：无语义的次要标签、禁用态、占位。

## 2. 基准尺度（排版与缩放）

- **`1rem = 18px` 为全局基准字号**（`base.css html{font-size:18px}`；终端 `.term-app{font-size:18px}` 与之统一）。
- 所有**排版尺寸**（字号、行高）一律使用 `rem`，随根字号统一缩放；**结构性间距**（侧栏/信息栏宽度、头部高度、内边距、网格间隙、卡片内距）按同一比例给定。
- 调整整体观感时改根字号与结构令牌即可全局生效，**不存在"放大某个局部"的临时操作**——新旧页面共享同一基准。
- 页面应在 1920×1080 下"刚好撑满"：内容不多时通过更大的字号/间距填充留白；文字过小不可辨认时通过提高基准解决。
- 内容区使用 `overflow-y:auto` 承载滚动；**放大或改版不得让本来没有滚动条的页面新增滚动条**，尤其严禁横向滚动条。

---

## 3. 终端信息显示顺序

- 显示「座位号 / 机器名 / IP 地址」三者时，**顺序恒为：座位 → 机器名 → IP**。
- 表单从上到下的字段顺序与之一致（先设座位、再设机器名、最后设 IP）。
- 例外：当某功能直接锚定其中一项时可调整（如"修改 IP"页可把 IP 列前置）。
- 终端在界面中以**座位号**指代（"座位 A01"或"A01"），不拼接机器名展示。当显示区域足够时，可补充"教室 + 座位"。

---

## 4. 标题区与按钮位置

- **带返回按钮（←）的标题区**：标题显示「当前所处的功能名」（如"网络同传""桌面管理""设置本机"），**不**显示教室名称（教室名不描述当前功能，可作次要信息附在功能名之后，弱化显示）。
- **返回按钮与取消按钮不同时出现**：进入子流程用"← 功能名"返回；弹框/独占流程内用"取消"。二者表达同一意图，不重复。
- **表单的保存按钮置于表单卡片外部、左下方**，不在视觉上被囊括进表单卡片内部，使"表单内容"与"提交动作"层次分明。
- **「返回 + 标题」行可右置按钮区**：当一页的核心操作只有少数几个（如桌面管理的"导入桌面""导出桌面"），将其放在标题栏最右侧，使中央内容区只保留主体（如桌面卡片），减少视觉噪声。

---

## 5. 表单呈现（对齐 / 宽度 / 不换行）

统一通过 `.prep-field` / `.prep-hint` / `.prep-section` 三件套实现，规则如下：

- **标签宽度统一**：`.prep-field label` 使用变量宽度 `--prep-label-w`（默认 96px），并 `white-space:nowrap` 保证标签自身不换行。需要更长标签的表单（如"数据盘大小 (GB)"）整体加大 `--prep-label-w` 并相应加宽容器，而非让标签换行。
- **输入框不溢出**：`.prep-field input/select` 设 `min-width:0` 且全局 `box-sizing:border-box`，使输入框在弹性布局中可收缩、永不超出表单宽度——**杜绝横向滚动条**。
- **提示/状态与输入框左对齐**：输入框下方的提示文字、连接状态框统一用 `.prep-hint`，其左缩进 `= 标签宽度 + 间距`（`calc(var(--prep-label-w) + 12px)`），与输入框左边缘严格对齐。
- **非必要不换行**：一行表单（标签 + 输入 + 说明）若因宽度不足而换行，应**加宽表单容器**使其容纳于一行；仅当容器宽度被刻意限制时（如网络同传左右分栏页）才允许内部换行。
- **分段标题**：表单分段（如设置本机的"教室与服务器/座位与机器名/网络配置"）每段都要有 `.prep-section` 标题，**第一段也不能省略标题**。

## 6. 弹框与独占流程

- **弹框（modal）**用于：轻量、一次性、不改变当前页面上下文的配置（如导入桌面参数、导出选择）。弹框宽度要能让其中最长的一行（标签 + 输入）**完整显示在一行内**，不可让标签或说明换行——宽度不足时加大弹框 `max-width`。
- **独占流程（in-page exclusive）**用于：步骤较多、需要专注的主流程（如平台侧导入终端清单、网络同传三步）。独占流程以"← 返回xxx"回到来源，**不叠加遮罩弹框**。
- **浮层菜单（下拉/溢出菜单）**必须能完整显示：当菜单位于滚动容器内、向上弹出会被上方区域（如标题栏）裁切时，应自动检测可用空间并在触发按钮上/下方翻转，确保始终完整可见。

---

## 7. 座位网格（终端可视化）

- 教室终端以**二维座位网格**呈现，替代抽象的终端列表/MAC 列表，把物理位置直观化。
- 座位需能表达三种布局状态——**可用 / 禁用 / 删除**，用于应对非规整教室布局：删除用于裁剪实际不存在的座位（如布局规整化后多出的格子、被柱子占用的位置），不参与编号与 IP 分配；禁用用于保留一个座位并占用编号 / IP 名额、本次不纳入，供后续补装设备；可用为正常参与分配。具体由何种操作在三态间切换不限定。
- 座位同时表达运行 / 任务态：选中态用强调色边框 + 浅底；已绑定 / 已完成用绿；离线 / 禁用用灰；警告用黄。
- 任务进行中，块内显示阶段（传输/写入/重启）；**完成态以对钩"✓"表示，不写"完成"二字**。
- 批量部署主流程内座位网格的过程 / 结果呈现与逐台确认见 [云桌面批量部署交互设计](./云桌面批量部署交互设计.md)。

---

## 8. 说明文字

- 说明文字（hint）一律弱化：小字号、`--*-text3` 弱文字色，紧贴其所解释的输入框下方并左对齐。
- 规则类说明应给出**实时预览**（如"机器名 = 前缀 + 座位号"配合实时拼接结果），而非要求用户自行心算。
- 不在说明文字里堆叠与当前操作无关的信息。

---

## 9. 导入 / 导出

- **导入**：拖拽/选择文件 → 系统读取内容 → 展开参数设置区 → 确认。平台侧导入采用「点击按钮展开独占区域」的模式（与"导入终端清单"一致），不使用遮罩弹框。
- **导出**：选择对象（多选，支持全选/取消）→ 选择目标文件夹（支持浏览或直接输入路径）→ 实时预览导出文件名 → 确认。**步骤合并为一屏**，不拆分多步骤提示。
- **路径输入**统一处理 Windows 与 Unix 风格（冒号、正反斜杠），并据此推断分隔符、给出跨平台预览。
- 导出内容随附必要的上下文：终端侧导出桌面包时附带本教室与本终端信息，便于线下导入还原（详见 业务事实层的终端导出/平台导入机制说明）。
- **导出按钮的妥协设计**：桌面管理的"导出桌面"设计为以统一格式一次性导出桌面信息与逐终端内容；本机信息随该导出一并携带，**不另设"导出本机信息"独立按钮**，避免两个含义相近的导出入口并列。单终端导出清单与网络同传全终端导出清单是两条不同粒度的入口，各自独立；教师终端单台部署后如需入服务器，由服务器侧单独导入。

---

## 10. 实时预览与即时反馈

- 凡"输入规则 → 生成结果"的场景，应在输入区旁（通常右侧或下方）给出实时预览，让用户边输入边看到结果，而非事后才知道。例：批量部署的布局 / 命名 / IP 规则输入后，右侧即时预览每个座位的机器名与 IP；导出文件名随参数实时拼接。
- 表单中依赖连通性的字段（如服务器地址），应在输入框失焦（blur）后自动发起连通性检测并回显状态，不迫使用户额外点击"检测"。
- 实时预览与即时检测不得打断当前输入：不因一次预览或检测而刷新整页、重置已填内容、或让滚动位置跳动（跳到页顶 / 页底）。

---

## 11. 图标与文案

- **不使用 emoji**（不专业）。功能性符号（如 `⚠` 警告三角、`✓` 对钩、`←` 返回箭头、`···` 溢出）可用。
- 中文标点统一；括号、单位（GB）等与数值同排不换行。
- 进度/结果文案精炼：用"成功 N · 失败 M / 共 K"的紧凑式，不啰嗦。
