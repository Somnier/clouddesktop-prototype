# 云桌面交互事实

> 文档角色：设计事实层的核心文件之一。
> 作用：沉淀云桌面原型在交互行为层面的既定事实——这些是多轮迭代验证后固化的交互方式，约束原型实现与后续设计。与「UI 呈现原则」（侧重视觉/排版）互补，本文侧重「行为/流程」。

---

## 1. 终端首页（设置本机）

- 首页为功能入口列表：设置本机 / 管理桌面 / 网络同传 / 替换故障终端 / 重置终端。
- **「设置本机」合并了原"设置本机"与"设置服务器"**，单页分三段：教室与服务器 / 座位与机器名 / 网络配置。
- 字段顺序（自上而下）：教室名称 → 服务器地址（带连通性检测）→ 座位号（起始字母 + 序号，实时预览）→ 机器名前缀（实时预览完整机器名）→ IP/子网掩码/网关/主 DNS/备用 DNS。
- **DNS 拆为主 DNS、备用 DNS 两行**，不再用逗号分隔单输入框。
- **服务器地址失焦自动检测连通性**：输入后失焦触发检测（spinner → 已连接/无法连接），单一计时器防止检测堆叠。
- 教室名称用于"同一教室约定"：物理同教室的终端应填相同教室名，平台据此归并（即便教室只有一台终端，如单独安装教师机，也可凭此导出含教室信息的单机清单）。

## 2. 桌面管理

- 页面元素精简为：标题栏（含剩余空间 + 导入/导出按钮）+ 桌面卡片网格。无容量饼图、无"未同步"横幅。
- 卡片"···"溢出菜单含：复制桌面 / 删除桌面 / 设为默认启动 / 隐藏桌面。菜单 `position:fixed` 锚定按钮，自动上/下翻转，不被标题栏裁切。
- 未同步状态与“同步到服务器”按钮用警告色（黄）；默认启动用中性灰标签。
- 导出桌面对话框随附本教室、本终端（座位 / 机器名 / IP）信息。

## 3. 网络同传（三步主流程）

- 三步：**① 布局设置 → ② IP 设置及进度 → ③ 传输桌面及进度**。三步以可点击长条呈现于标题下方，**用户可自由切换至任意一步**（去除强制 gating）。
- **座位网格居左、输入区居右**。
- **第一步（布局设置）**：用户只关心行列/起点/机器名前缀/座位起始字母/教室名。后台同时在扫描发现局域网新终端并隐式维护接管列表，但页面**不显示扫描状态、不显示超容量告警**。
- **第二步（IP 设置）**：进入即接管，所有终端转为受控态。为布局每个座位设 IP；受控终端在各自屏幕上按回车 / 空格键，按布局方向顺序依次绑定座位并同步对应 IP；点“下一步”固化绑定与地址。母机在左侧座位网格实时查看绑定进度。
- **第三步（传输桌面）**：执行部署。
- 第二、三步各含三子态：**设置参数 / 进度 / 结果**。**不再叠加"上次部署结果"层**。
- 传输完成后座位视图显示结果，**完成态用对钩"✓"**；用户恢复点击能力，**点选座位可取消选择，使下一次传输跳过该座位**。
- 该页仅保留"导出终端清单"与"结束管理"两个动作按钮（标题栏右侧）。
- 标题显示功能名"网络同传"，教室名作次要信息弱化附带。

## 4. 故障处理

- "替换故障终端""重置终端"为首页一级入口，教室座位网格选座后一键执行，进度实时反馈。

## 5. 平台侧关键交互

- 三栏布局：左侧导航 + 中央内容 + 右侧信息面板。
- **导入终端清单**：上传清单后按教室名自动匹配/归并教室，并据清单内桌面包一并导入平台桌面资产（省去单独导入桌面）。
- **桌面资产**：按桌面名跨教室聚合；无教室引用的桌面不可点击溯源、标注"无教室引用"；2+ 教室引用与 1 教室引用数据并存。
- **服务器地址变更**：左侧输入规则 + 推送，右侧持久化展示上次变更结果（切走再切回仍可见），提供清除历史。
- **导入/导出**采用点击展开的独占区域（与终端侧导入终端清单一致），非遮罩弹框。
- 总览日志：来源终端以座位号可点击跳转终端详情；告警右栏点击座位跳转终端（非教室），hover 显示告警详情，可两行展示（教室 + 告警）。

## 6. 同步与渲染

- 四端（导演台 / 母机 / 受控 / 平台）共享一份 JSON 状态，经 SSE（约 600ms tick）实时同步。
- 渲染采用状态指纹比对 + 输入框 / 滚动位置保持，避免 SSE 重渲染打断用户输入与滚动。
- 规则类输入在失焦 / 变更时（防抖）保存，不在每次按键即重渲染，以免打断输入。

## 7. 演示与产品分离

- 产品界面（母机 / 受控 / 平台）是可交付形态，不携带任何演示 / 模拟性按钮；现场真实发生的动作在产品中由真实终端触发。
- 演示 / 模拟由独立的“导演台”承担，其定位、职责与边界见 [云桌面导演台设计事实](./云桌面导演台设计事实.md)。
