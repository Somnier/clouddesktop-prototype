# 云桌面原型迭代取舍

> 文档角色：04_云桌面设计事实 的核心文件之一。
> 作用：记录云桌面原型从 09 评审版 → 10 简化版 v1 → 11 简化版 v2（现 07_云桌面原型）的演进，以及"哪些做、哪些不做"的范围取舍。设计文档（05 终端侧、06 平台侧）保留了前期完整设计，原型按本文取舍实现；设计与实现的差异以本文为准，属正常的功能实现取舍，不视为缺陷。

---

## 1. 三个版本的演进主线

| 版本 | 主题 | 关键决策 |
|---|---|---|
| **09 评审版**（已归档移除） | 功能广度 | 完整实现所有流程，含考试、终端自检、资产同步详情；部署用 5 步列表式向导；以终端列表呈现教室。 |
| **10 简化版 v1**（已归档移除） | 交互收敛 | 隐去考试、自检等辅助流程，聚焦核心运维；以**座位网格**替代终端列表（引入 `grid` + `bindings` 数据结构）；部署向导 5 步 → 4 步。 |
| **11 简化版 v2**（现 **07_云桌面原型**） | 操作连续性 | 进一步合并：接管与工作台合并为「网络同传」单流程；「设置本机」与「设置服务器」合并；网络同传重构为可自由切换的**三步主流程**（布局设置 / IP 设置及进度 / 传输桌面及进度）；SSE 渲染加入状态指纹与输入/滚动保持，消除同步抖动；平台侧三栏布局。 |

> 注：09、10 目录已从仓库移除（保留于 Git 历史）。当前仓库仅保留 07_云桌面原型 作为唯一原型蓝图。

## 2. 做什么（核心范围）

- **可视化教室管理**：用二维座位网格映射物理座位，替代 MAC/列表抽象。
- **零跳转主流程**：把"扫描发现 → 命名/IP 规则 → 座位绑定 → 桌面传输"收敛进一个可自由切换的工作台，减少页面跳转与上下文丢失。
- **故障即时恢复**：把"一键替换""一键重置"放在终端首页一级入口，应对现场应急。
- **平台单屏概览**：Web 侧提供仪表盘（健康度、服务器监控、告警日志）三栏布局的"单一视图"。
- **终端信息一致性**：座位/机器名/IP 在两端按统一口径与顺序呈现；桌面资产由终端上报汇总。
- **导出—导入闭环**：终端导出（单机或整教室）→ 平台按教室名归并、按清单自动导入桌面包（详见 03 业务事实）。

## 3. 不做什么（刻意搁置）

以下能力在早期设计/09 版本中存在，简化版**刻意不实现独立页面与流程**，以聚焦核心运维：

| 搁置项 | 早期状态（09） | v2（07_云桌面原型）现状 |
|---|---|---|
| **考试流程** | 含 `exam-main / exam-desktop / exam-confirm / exam-progress / exam-active` 等多屏 | 不实现。首页无考试入口，无对应屏处理。设计文档 05 的 TERM-07 保留考试桌面流程设计供后续参考。考试桌面在 `seed.json` 中仅作为一种桌面数据类型存在。 |
| **终端自检流程** | 独立 `selftest` 屏（"本机测试"入口） | 不实现。首页与导航均移除入口。 |
| **资产同步进度详情** | 独立 `asset-sync` 监控屏 | 不实现。改为后台自动同步 + 桌面卡片上的状态 pill（未同步/已同步）表达，不再有独立进度页。 |
| **冗余确认屏** | `maintConfirm / examConfirm` 等独立确认页 | 不实现。改为就地确认或座位网格反馈。 |
| **多步部署向导** | 5 步列表式向导 | 收敛为网络同传三步主流程，且三步可自由切换、就地展示参数/进度/结果，不再叠加"上次部署结果"层。 |

> 这些搁置属于"评审/原型阶段的功能实现取舍"。设计层（05/06）保留完整设计描述；当某能力进入实现排期时，可基于设计文档恢复，并在本文更新其状态。

---

## 4. 设计与实现的对齐约定

1. **设计文档（05 终端侧 / 06 平台侧）是完整设计的载体**，包含已实现与暂未实现的页面。
2. **07_云桌面原型是当前实现蓝图**，其覆盖范围以本文第 2、3 节为准。
3. 当设计文档描述的某页面/流程在原型中未实现时，以本文标注的"搁置/不做"为准，不应被误判为实现缺失。
4. 新增或恢复某搁置能力时：先在设计文档补齐/确认设计 → 在原型实现 → 回到本文更新"做/不做"状态。
