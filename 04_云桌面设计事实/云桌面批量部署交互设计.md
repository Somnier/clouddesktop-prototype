# 云桌面批量部署交互设计

> 文档角色：设计事实层的核心文件之一。
> 作用：约定母机批量部署（网络同传）主流程的界面呈现与交互原则。业务流程与数据变化见业务事实层的 [云桌面单教室批量部署机制说明](../03_云桌面业务事实/云桌面单教室批量部署机制说明.md)；本文只约定"怎样呈现、怎样交互"。
> 原则：以下是指导思想，给出状态与意图即可，具体控件与操作方式由页面设计自由实现，只要满足这些事实。

---

## 1. 座位网格

- 教室终端以二维座位网格呈现，把物理位置直观化，替代抽象的终端列表 / MAC 列表。
- 网格的每个座位需要能表达三种布局状态：**可用、禁用、删除**。
  - 删除：该位置实际没有座位（如布局规整化后多出的格子、被柱子占用的位置），不参与编号与 IP 分配。
  - 禁用：保留该座位并占用一个编号 / IP 名额，但本次不纳入，供后续补装设备。
  - 可用：正常参与布局与分配。
- 座位需提供在这三种状态间切换的途径，具体由何种操作触发不限定。
- 座位还需表达运行 / 任务态：选中、已绑定 / 已完成、离线 / 不可用、警告等，用颜色与标记区分（语义色见 [云桌面UI呈现原则](./云桌面UI呈现原则.md)）。

## 2. 多步主流程

- 批量部署的布局、IP 设置、桌面传输三个阶段在同一功能内以多步呈现，步骤导航常驻、可自由切换到任意一步，不强制单向推进。
- 标题区显示功能名（如"网络同传"），教室名作次要信息弱化附带。
- 该功能内只保留少量动作按钮（如导出终端清单、结束管理），其余交由步骤内的主操作。

## 3. 同一流程内表达过程与结果

- 设置参数、执行进度、执行结果是同一步骤内的不同子态，**就地切换**，不另设独立的"结果页"或"收口态"中转。
- 不叠加"上次结果"这类历史层；当前步骤只呈现当前这一轮的参数 / 进度 / 结果。
- 进度与完成在座位网格上逐位呈现：进行中显示阶段，完成以对钩"✓"表示而非"完成"二字。

## 4. 逐台确认的呈现

- 批量规则展开为逐台写入结果后，以座位为单位逐位呈现绑定与地址、传输进度与结果，供用户核对。
- 结果态允许用户调整下一轮的纳入范围（例如取消某座位使下一轮跳过它）；具体交互方式不限定。
- "保持现状"的终端需明确标为"本次未纳入"，并回显其沿用的当前内容，不呈现为掉线或失败。

## 5. 终端信息显示顺序

- 显示「座位号 / 机器名 / IP」三者时，顺序恒为：座位 → 机器名 → IP；表单字段顺序与之一致。
- 例外：功能直接锚定某项时可调整（如"修改 IP"页可把 IP 前置）。
- 终端以座位号指代（"座位 A01"或"A01"），不拼接机器名；区域足够时可补"教室 + 座位"。

## 6. 可勾选列表

- 凡需人工勾选终端 / 桌面 / 教室的列表，应支持排序与分页；可排序字段、默认排序与分页策略按对象类型与数据量分别确定，不统一写死。
