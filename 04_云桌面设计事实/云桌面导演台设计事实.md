# 云桌面导演台设计事实

> 文档角色：设计事实层的核心文件之一。
> 作用：约定原型中"导演台"这一演示控制界面的定位与职责。导演台不是产品的一部分，而是为演示与模拟服务的独立界面；本文把它与产品界面（母机 / 受控 / 平台）的边界写清楚。

---

## 1. 导演台是演示界面，不是产品功能

- 产品界面（母机 / 受控 / 平台）是可交付形态，**不携带任何演示 / 模拟性按钮**。现场真实发生的动作——受控终端按回车绑定座位、终端开关机、硬件故障——在产品中由真实终端、真实现场触发。
- 导演台是演示与模拟的唯一控制面，独立于产品界面而存在，用于在没有真实机房的情况下推演四端联动。
- 因此产品界面里不应出现"模拟绑定""模拟开机""制造故障"这类入口；凡需要"替现场发生一件事"的操作，都收敛到导演台。

## 2. 导演台的两项职责

### 2.1 代替现场动作

为演示触发那些真实环境下由现场产生、原型中无真机可触发的事件：

- 模拟受控终端按回车 / 空格键绑定座位。
- 切换当前聚焦的教室 / 终端。
- 模拟终端开机 / 关机。
- 模拟硬件故障。
- 导航跳转到各端的各个功能页，便于演示按剧本推进。

### 2.2 可视化共享数据

- 导演台可视化四端共享的同一份数据，便于演示者观察当前全局状态。
- 它读取与产品界面相同的那一份状态，不另造一套数据。

## 3. 与产品界面的边界

- 导演台触发的动作，效果等同于现场真实发生：受控终端在导演台被"按回车"后，其绑定结果与真实按键一致地反映到母机 / 平台。
- 产品界面只呈现这些动作产生的真实结果，不暴露触发它们的演示入口。
- 在真实部署中导演台不存在；产品界面的行为不依赖导演台，导演台只在演示场景叠加于产品之上。
