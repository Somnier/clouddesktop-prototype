# 云桌面设计事实索引

> 文档角色：设计事实层的索引页。
> 作用：登记本层全部设计事实文档，并约定"设计事实如何写、写什么、何时写"。本层沉淀界面呈现与交互层面的设计事实——业务事实约束"系统是什么、机制如何"，设计事实约束"界面如何呈现、交互如何进行、范围做到哪"。二者并列，共同作为页面设计与原型实现的认知底座。

## 1. 本层放什么

满足以下任一条件的结论，应沉淀为设计事实：

- **跨页面的呈现规则**：颜色语义、排版基准、信息显示顺序、表单 / 弹框 / 按钮的统一规范。
- **跨页面的交互方式**：固化的操作流程与控件行为。
- **范围取舍**：哪些能力做、哪些不做。

## 2. 设计事实怎么写、何时写

1. **何时写**：在推进设计、修改实现的同时同步沉淀或校对设计事实。设计事实不只来自业务概念，还须结合界面与交互的最终形态——发现既有设计事实不足或与认知不符时，立即回到本层补全 / 订正。
2. **自上而下推演式修改，不打补丁**：设计事实表达当前成立的完整规则，不体现"某次改了什么、为什么改"。修订时重写相关章节使其自洽，严禁以"历史上曾用 X，已改为 Y""重要：本次强调 Z"等补丁式、强调式语句把单次修改与修改原因写进事实。读者只应读到当前规则。
3. **重写即正名**：文档被实质性重写、主题或范围已变时，文件名随内容同步更名。
4. **概念锚定统一口径**：指代同一对象时一律沿用 [核心概念与统一口径](../03_云桌面业务事实/云桌面核心概念与统一口径.md) 的术语与口径。

## 3. 文件分组

- [云桌面UI呈现原则.md](./云桌面UI呈现原则.md)：设计令牌（颜色 / 基准）、语义色约定、排版尺度、信息显示顺序、标题与按钮位置、表单对齐与宽度、弹框与独占流程、座位网格、说明文字、导入导出、图标文案。

### 3.2 交互方式
- [云桌面交互事实.md](./云桌面交互事实.md)：终端首页 / 设置本机、桌面管理、网络同传三步主流程、故障处理、平台侧关键交互、同步与渲染、演示与产品分离。

### 3.3 范围取舍
- [云桌面原型范围取舍.md](./云桌面原型范围取舍.md)：原型做什么、不做什么，以及当前未实现的能力清单。

## 4. 与其它层的衔接

1. **业务事实** 约束机制与口径；设计事实约束呈现与交互。二者并列，共同作为页面设计与原型实现的认知底座。
2. **终端侧 / 平台侧页面设计** 引用业务事实与设计事实取口径。
3. **原型** 是可运行实现；其早期可行性验证随原型一并归档。
